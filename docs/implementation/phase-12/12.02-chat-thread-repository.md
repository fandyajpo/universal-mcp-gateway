## Goal

Implement the Mongoose schema and `TenantAwareRepository` for chat threads and messages.

## Business Context

Chat data must be persisted, retrieved, and queried efficiently. Threads are the primary organizational unit for conversations and need workspace-scoped access control.

## Technical Context

The repository extends `TenantAwareRepository` to ensure workspace isolation. Threads and messages are stored in separate collections with appropriate indexes for fast retrieval.

## Requirements

- Mongoose schema for Thread: workspaceId (indexed), title, agentId, participantIds, createdAt, updatedAt, deletedAt
- Mongoose schema for Message: threadId (indexed), role, content, toolCalls, toolResults, citations, metadata, createdAt
- Indexes: threadId + createdAt for chronological listing, workspaceId for tenant isolation, agentId for agent-scoped queries
- ThreadRepository: create, findById, findByWorkspace, update, softDelete, listByWorkspace (paginated)
- MessageRepository: create, findByThread (paginated, chronological), deleteByThread
- Soft delete support on threads (not messages — they are immutable)
