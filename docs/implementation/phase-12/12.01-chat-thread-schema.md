## Goal

Define and implement the Zod validation schemas for chat threads and messages in `@repo/validation`.

## Business Context

Chat is the primary user interface of the platform. Threads organize conversations between users and AI agents. Messages contain user prompts, AI responses, tool calls, and citations.

## Technical Context

The chat domain requires two core entities: Thread (a conversation container) and Message (individual turns). Threads belong to a workspace and optionally to an agent. Messages belong to a thread and contain the actual conversation data.

## Requirements

- Thread schema: id, workspaceId, title, agentId (optional), participantIds, createdAt, updatedAt, deletedAt
- Message schema: id, threadId, role (user/assistant/tool), content, toolCalls, toolResults, citations, metadata, createdAt
- Validation for creating threads: title required, optional agentId
- Validation for sending messages: content required for user role, toolCalls for tool role
- Pagination validation for thread history queries
