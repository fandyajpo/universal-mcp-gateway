## Goal

Integrate MCP tool execution into the chat flow, allowing AI to invoke tools during conversations and display tool interactions in the UI.

## Business Context

MCP tools extend the AI's capabilities beyond text generation. Tools can search documents, query databases, call external APIs, and perform actions. The chat UI must surface tool usage transparently so users understand what the AI is doing.

## Technical Context

The AI Gateway calls tools through the MCP Gateway. Tool execution can be synchronous (fast tools during response generation) or asynchronous (long-running tools that report results later). The streaming protocol includes tool_call and tool_result events.

## Requirements

- Tool execution during AI response generation
- Display tool call in progress (spinner, tool name, input parameters)
- Display tool result (success/error, output preview)
- Support for sequential multi-tool calls (chain tools)
- Support for parallel tool calls (fan-out)
- Tool execution timeout handling (30s default)
- Tool error handling in UI (show error, allow retry)
- Tool confirmation for destructive actions
- Tool usage history in thread metadata
