## Goal

Implement the streaming chat API route that delivers AI responses as Server-Sent Events (SSE) with real-time tool execution updates.

## Business Context

The chat experience must feel responsive. Streaming allows users to see AI output as it's generated, including intermediate tool calls and RAG citations, rather than waiting for the complete response.

## Technical Context

The API route uses Next.js streaming response (ReadableStream) to send SSE events. The chat service must support an async generator pattern that yields events for: text tokens, tool calls, tool results, citations, errors, and completion.

## Requirements

- POST `/api/v1/chat/:threadId/messages` with streaming response
- SSE event types: `token` (text delta), `tool_call`, `tool_result`, `citation`, `error`, `done`
- Request body: `{ content: string, threadId: string }`
- Response: `text/event-stream` content type
- Backpressure handling for slow consumers
- Abort signal support for user cancellation
- Timeout after 60 seconds of no response
- Error events for AI failures (don't crash the stream)
- Rate limiting: 10 requests per minute per user

- SSE stream delivers events in correct order
- Text tokens stream incrementally
- Tool calls and results interleaved in stream
- Citations included in stream
- Abort cancels the AI request
- Timeout produces error event
- Rate limit returns 429
