## Goal

Build the message composition area with text input, file attachments, send button, keyboard shortcuts, and multi-line editing.

## Business Context

The message input is the primary interaction point for users. It must support rich text entry, file uploads (PDFs, images), drag-and-drop, keyboard shortcuts, and graceful handling of long messages.

## Technical Context

This is a Client Component using React Hook Form for input management. File uploads use the R2 signed URL flow from Phase 05. The input supports `Shift+Enter` for newlines and `Enter` to send.

## Requirements

- Multi-line text input with auto-resize (min 2 lines, max 12 lines)
- Send on Enter (configurable), newline on Shift+Enter
- File attachment button with type validation (PDF, image, text)
- Drag-and-drop file attachment zone
- Attachment preview with remove button
- Upload progress indicator
- Send button (disabled when empty or uploading)
- Keyboard shortcut: `Cmd+K` for command palette, `Cmd+Enter` to send
- Character counter at 4000 chars with warning at 3000
- Paste handling: plain text by default, files for attachment
- Disabled state while AI is responding
