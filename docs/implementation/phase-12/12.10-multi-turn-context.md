## Goal

Implement conversation context management that maintains coherent multi-turn conversations by managing the context window, summarizing past messages when needed, and preserving relevant information across turns.

## Business Context

AI conversations span many turns. Without context management, the conversation window grows indefinitely, consuming tokens and diluting focus. The system must intelligently manage context: keep recent messages, summarize older ones, and preserve key facts.

## Technical Context

Context management happens in the chat service layer. The context builder from the RAG engine (Phase 08) is extended to handle conversation history. Token counting determines when summarization is needed.

## Requirements

- Token-aware context window management (configurable limit, default 8K tokens)
- Strategy: keep last N messages verbatim, summarize older segments
- Summarization trigger: when total tokens exceed 70% of window
- Key fact extraction and preservation across summarization
- System prompt includes workspace context, agent configuration, and user preferences
- RAG results injected into context at appropriate positions
- Token usage tracking per conversation turn
- Context window visualization in debug mode
- Configurable per-thread context window size
