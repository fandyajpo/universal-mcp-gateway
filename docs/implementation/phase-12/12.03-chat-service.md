## Goal

Implement the chat service that orchestrates message processing, AI responses, MCP tool execution, RAG retrieval, and thread management.

## Business Context

The chat service is the core business logic of the AI workspace. It coordinates between the AI Gateway (for LLM responses), the MCP Gateway (for tool execution), the RAG Engine (for knowledge retrieval), and the database (for persistence).

## Technical Context

The service follows Clean Architecture: it depends on repository interfaces, the AI Gateway client, the MCP gateway client, and the RAG engine. It does NOT directly access the database or external APIs.

## Requirements

- `createThread(input, userId)`: Creates a new thread with workspace context
- `sendMessage(threadId, content, userId)`: Processes a user message, generates AI response, executes tools, persists conversation
- `getThread(threadId, userId)`: Retrieves a thread with access check
- `listThreads(workspaceId, pagination)`: Paginated thread listing
- `getMessages(threadId, pagination)`: Paginated message history
- `deleteThread(threadId, userId)`: Soft delete with permission check
- Message processing pipeline: validate → persist user message → build context → call AI → execute tools → persist AI response → return
- Context building: include conversation history, RAG results, and system prompt
