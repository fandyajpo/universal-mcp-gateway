## Goal

Run the complete verification suite for Phase 13: Admin. Execute all tests, verify type safety, lint compliance, build integrity, and perform full manual end-to-end testing of every admin feature. Create integration tests that verify the interaction between steps, and produce a verification report.

---

## Business Context

The admin phase covers security-critical functionality: user management, audit logs, feature flags, and system health. A failure in any of these areas could have serious consequences — unauthorized access, data loss, or undetected system degradation. Comprehensive verification is essential before moving on to Phase 14. This step ensures that all admin features work correctly individually and together.

---

## Technical Context

Verification follows the project's standard verification pattern established in Step 00.15. It includes: `pnpm test` (unit + integration), `pnpm typecheck`, `pnpm lint`, `pnpm build` for the admin app. Manual verification follows a documented test script that covers all acceptance criteria and edge cases. Integration tests verify cross-step interactions, such as the auth guard correctly protecting all admin routes and the audit log correctly recording mutations from other admin steps.

## Requirements

### Functional requirements

- Run all unit tests from steps 13.01-13.08 and verify 100% pass rate
- Run all integration tests and verify API behavior end-to-end
- Verify TypeScript strict mode with no errors across the admin app and all its dependencies
- Verify lint passes with no errors or warnings
- Verify admin app builds successfully in production mode
- Execute manual test script covering all admin features
- Verify cross-step integration: auth guard protects all admin pages
- Verify audit events are created for mutations from all admin steps
- Test error states, loading states, and empty states for all admin pages
- Verify responsive layout at 1024px, 768px, and 375px breakpoints

### Non-functional requirements

- Test suite must complete in under 10 minutes
- Manual verification must be documented with expected vs actual results
- Test coverage must meet or exceed 80% across all new code
- No failing tests, type errors, lint errors, or build errors are acceptable

### Performance requirements

- Unit tests: < 30s total
- Integration tests: < 5 minutes total
- Type check: < 2 minutes
- Lint: < 1 minute
- Build: < 3 minutes

### Security requirements

- Verify that unauthenticated requests to all admin routes return 401
- Verify that authenticated non-admin requests return 403
- Verify that audit logs correctly capture all admin mutations
- Verify that impersonation is properly logged
- Verify that feature flag changes are audited

### Accessibility requirements

- Quick accessibility check of all admin pages using axe-core or similar tool
- Verify keyboard navigation through all interactive elements
- Verify screen reader announcements for dynamic content

## Architecture Constraints

- Verification must be repeatable — the manual test script must document exact steps and expected results
- Integration tests must use a test database (not production)
- All tests must be independent — running them in any order must produce the same results
- Manual verification must include at least one positive and one negative test for each feature
- Results must be documented and any failures must be fixed before proceeding
