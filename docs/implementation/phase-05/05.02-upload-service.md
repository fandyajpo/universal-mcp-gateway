# Step

Step ID: 05.02

Step Name: File Upload Service

Estimated Time: 1.5 hours

Priority: Critical

Difficulty: Medium

Dependencies: 05.01

---

## Goal

Implement the file upload service that handles multipart file uploads, streams files to R2, creates document records in MongoDB, and returns file metadata. The service supports both single and batch uploads with configurable concurrency.

---

## Requirements

1. Create `packages/storage/src/upload-service.ts`:
   - `uploadFile(buffer, filename, contentType, workspaceId, userId, entity?)` — validates file, uploads to R2, creates Document record in MongoDB, returns file metadata.
   - `uploadFiles(files, workspaceId, userId)` — batch upload with configurable concurrency (default: 3 parallel).
   - `deleteFile(fileKey, workspaceId)` — deletes from R2 and marks Document as deleted.
   - `getFile(fileKey, workspaceId)` — returns file metadata and generates download URL.
2. Upload flow:
   1. Validate file type (via `@repo/storage` type validator)
   2. Validate file size against workspace limits
   3. Generate storage key: `{workspaceId}/{entity}/{date}/{uuid}-{filename}`
   4. Upload to R2 with content-type and metadata (workspaceId, userId, originalName)
   5. Create Document record in MongoDB via DocumentRepository
   6. Return `{ id, key, filename, size, contentType, url }`
3. Error handling: R2 upload failure removes the MongoDB record (compensating transaction).

---

## Files To Create

---

## Acceptance Criteria

1. Single file upload succeeds and returns file metadata.
2. Batch upload processes files concurrently.
3. Upload failure cleans up MongoDB record.
4. Duplicate filename handling: appends UUID prefix to prevent collisions.
5. Document record has correct workspaceId, userId, and metadata.
