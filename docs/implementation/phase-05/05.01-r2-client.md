# Step

Step ID: 05.01

Step Name: R2 Client Setup

Estimated Time: 1 hour

Priority: Critical

Difficulty: Easy

Dependencies: 00.02, 00.03

---

## Goal

Create the `@repo/storage` package with a configured Cloudflare R2 client using the AWS S3 SDK (`@aws-sdk/client-s3`). The client supports all required operations: putObject, getObject, deleteObject, listObjects, and signed URL generation.

---

## Requirements

1. Create `packages/storage/` with `package.json` (depends on `@repo/types`, `@repo/logger`, `@aws-sdk/client-s3`, `@aws-sdk/s3-request-presigner`).
2. Create `src/client.ts` — R2 client factory:
   - Reads config: `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET_NAME`, `R2_ACCOUNT_ID`, `R2_PUBLIC_URL` from `@repo/config`.
   - Creates `S3Client` with R2 endpoint: `https://{accountId}.r2.cloudflarestorage.com`.
   - Region: `auto` (R2 uses a global endpoint).
3. Create `src/storage-service.ts` with methods:
   - `upload(key, body, contentType, metadata?)` — streams file to R2
   - `download(key)` — streams file from R2
   - `delete(key)` — deletes a file
   - `list(prefix)` — lists files with given prefix
   - `exists(key)` — checks if a key exists (headObject)
   - `getMetadata(key)` — returns file metadata
4. Create `src/index.ts` barrel file.
5. Create `src/index.test.ts` (mocked S3 client).

## Acceptance Criteria

1. R2 client connects to Cloudflare R2 (integration test with real R2 bucket — optional, skipped if env vars not set).
2. Upload/download round-trip works for small files (<1MB).
3. Delete removes the file (headObject returns 404 after delete).
4. `list` returns files with the given prefix.
5. Unit tests pass with mocked S3 client.
6. `pnpm typecheck` and `pnpm lint` pass.
