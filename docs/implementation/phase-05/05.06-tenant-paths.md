# Step

Step ID: 05.06

Step Name: Tenant-Isolated Storage Paths

Estimated Time: 30 minutes

Priority: Critical

Difficulty: Easy

Dependencies: 05.01

---

## Goal

Implement a consistent, tenant-isolated storage path scheme for all files stored in R2. Every file key is prefixed with the workspace ID to ensure logical data isolation within the shared R2 bucket.

---

## Requirements

1. Path format: `/{workspaceId}/{entity}/{year}-{month}-{day}/{uuid}-{sanitizedFilename}`.
   - Example: `ws_abc123/documents/2026-06-10/a1b2c3d4-report.pdf`
   - `sanitizedFilename`: lowercase, spaces → hyphens, special chars removed.
2. Implement `generateStorageKey(workspaceId, entity, filename)`:
   - Takes workspaceId, entity type, original filename.
   - Generates UUID v4 for uniqueness.
   - Returns the formatted key string.
3. Implement `parseStorageKey(key)`:
   - Parses key into `{ workspaceId, entity, date, uuid, originalName }`.
   - Used for audit and retrieval.
4. Entity types: `documents`, `avatars`, `attachments`, `exports`, `connectors`, `temp`.
5. Implement `listWorkspaceFiles(workspaceId, entity?, prefix?)` — lists all files for a workspace using R2 `listObjects` with prefix filter.
6. Ensure path traversal prevention: `sanitizedFilename` cannot contain `..` or `/`.

---

## Files To Create

- packages/storage/src/paths.ts

---

## Acceptance Criteria

1. `generateStorageKey("ws_1", "documents", "My Report.pdf")` returns `ws_1/documents/2026-06-10/{uuid}-my-report.pdf`.
2. `parseStorageKey` correctly extracts all components.
3. Characters like `..`, `/`, `\`, null bytes are removed from filename.
4. `listWorkspaceFiles("ws_1")` only returns files for that workspace.
5. Two different workspaces have no overlapping file keys.
