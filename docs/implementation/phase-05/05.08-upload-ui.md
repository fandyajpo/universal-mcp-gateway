## Goal

Build a reusable file upload UI component with drag-and-drop zone, file preview, progress bar, error states, and file list. The component uses the upload API routes and provides a consistent upload experience across the web app.

---

## Requirements

1. Create `apps/web/src/components/ui/file-upload.tsx`:
   - Drag-and-drop zone with dashed border, "Drop files here or click to browse" text.
   - Click to open file browser (accepts configured MIME types).
   - Multiple file selection support.
   - File preview: shows icon based on type (PDF icon, image thumbnail, document icon).
   - Upload progress: individual progress bar per file + overall progress.
   - Success state: green checkmark, file name, size.
   - Error state: red border, error message (type invalid, too large, upload failed).
   - Retry button for failed uploads.
   - File list: shows all selected files with their current status (pending, uploading, success, error).
   - Remove button to deselect a file before upload starts.
2. Create `src/hooks/use-file-upload.ts`:
   - Manages upload queue with concurrency control (max 3 parallel).
   - Tracks progress per file.
   - Handles retry logic.
   - Returns `{ files, upload, cancel, retry, progress, isUploading }`.
3. Create `src/components/ui/file-icon.tsx` — renders file type icon (PDF, image, document, code, archive).
4. Create `src/components/ui/file-preview.tsx` — thumbnail preview for images, icon for other types.
5. Accessibility: keyboard navigation (Tab to zone, Enter/Space to browse), ARIA labels, focus management, screen reader announcements for upload progress.
6. Dark mode support.
7. Responsive: full-width on mobile, max-width on desktop.

## Acceptance Criteria

1. Drag-and-drop zone accepts files, shows visual feedback on hover.
2. Clicking zone opens file browser filtered by allowed types.
3. File preview shows correctly based on type.
4. Upload progress bars update in real-time.
5. Successful upload shows green state with file metadata.
6. Invalid file type shows error with clear message.
7. Oversized file shows size limit error.
8. Upload failure shows error with retry button.
9. Multiple files upload with controlled concurrency.
10. Keyboard accessible: Tab, Enter, Space, Escape.
11. Screen reader announces "Upload complete" and error states.
12. Dark mode renders correctly.
