# Step

Step ID: 05.05

Step Name: Size Limits and Enforcement

Estimated Time: 30 minutes

Priority: High

Difficulty: Easy

Dependencies: 05.04

---

## Goal

Implement file size limits per workspace tier and enforce them at upload time. Limits are stored in the workspace settings and cached in Redis. Exceeding the limit returns a 413 Payload Too Large error before any bytes are written to R2.

---

## Requirements

1. Default size limits per tier: Free (10MB per file, 1GB total), Pro (50MB per file, 50GB total), Enterprise (500MB per file, 1TB total).
2. Limits checked before upload: reject if file size > per-file limit.
3. Total usage tracked: increment workspace storage used on upload, decrement on delete. Enforce total storage limit.
4. Limits configurable per workspace via workspace settings (Enterprise only).
5. Usage stats: `getStorageUsage(workspaceId)` returns `{ used, limit, percentage }`.
6. Size validation in Zod: upload schema rejects files exceeding limit.
7. Error response: 413 with `{ error: "file_too_large", maxSize, actualSize }`.

---

## Files To Create

- packages/storage/src/validation/size-limits.ts
- packages/storage/src/usage-tracker.ts

---

## Acceptance Criteria

1. Uploading a file larger than the workspace limit returns 413.
2. Uploading a file within the limit succeeds.
3. Total storage usage increments on upload, decrements on delete.
4. Exceeding total storage limit blocks new uploads.
5. Limits can be overridden per workspace (Enterprise tier).
6. Usage stats are accurate.
