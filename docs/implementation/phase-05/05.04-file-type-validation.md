## Goal

Implement file type validation using magic bytes (the first few bytes of a file that identify its true format) rather than relying on file extension or MIME type from the HTTP request. This prevents MIME-type spoofing attacks where a malicious file is disguised as a different type.

---

## Requirements

1. Create `packages/storage/src/validation/file-type.ts`:
   - Magic byte signatures for allowed types: PDF (`%PDF`), PNG (`‰PNG`), JPEG (`ÿØÿ`), GIF (`GIF8`), WebP (`RIFF....WEBP`), SVG (`<svg` or `<?xml`), CSV/TSV (text-based), JSON (`{` or `[`), Markdown (text-based), ZIP/PK (`PK`), Excel (`PK` with specific offset), Word (`PK` with specific offset).
   - `validateFileType(buffer, allowedTypes?)` — reads first 512 bytes, matches against magic byte signatures, returns `{ valid: boolean, detectedType: string }`.
   - `getAllowedTypesForEntity(entity)` — returns allowed types per entity (e.g., documents: PDF, images; connectors: JSON, ZIP).
2. Create `packages/validation/src/schemas/storage.ts`:
   - Zod schema for file metadata: `fileTypeSchema`, `fileSizeSchema`, `uploadOptionsSchema`.
3. Allowed types configuration: environment variable `ALLOWED_FILE_TYPES` (comma-separated) or per-workspace setting.

## Acceptance Criteria

1. `validateFileType(Buffer.of("%PDF..."))` returns `{ valid: true, detectedType: "pdf" }`.
2. `validateFileType(Buffer.of("GIF8..."))` with `allowedTypes: ["pdf"]` returns `{ valid: false, detectedType: "gif" }`.
3. Renaming a `.exe` to `.pdf` is detected as invalid (magic bytes don't match).
4. Text-based types (JSON, CSV, Markdown) are detected by content inspection.
5. Validation completes in under 5ms for files up to 512 bytes.
