# Step

Step ID: 05.07

Step Name: Upload API Routes

Estimated Time: 1 hour

Priority: High

Difficulty: Medium

Dependencies: 05.02, 05.05

---

## Goal

Create Next.js API routes for file upload and download. The upload route accepts multipart form data, validates type and size, calls the upload service, and returns file metadata. The download route generates and redirects to a signed URL.

---

## Files To Create

- apps/web/src/app/api/files/upload/route.ts (POST — upload single/multiple files)
- apps/web/src/app/api/files/[fileId]/route.ts (GET — download redirect, DELETE — delete file)
- apps/web/src/app/api/files/[fileId]/metadata/route.ts (GET — file metadata)
- apps/web/src/app/api/files/storage/route.ts (GET — storage usage stats)
- apps/web/src/actions/files/upload.ts (Server Action for direct upload)
- apps/web/src/lib/file-upload.ts (client-side upload helper)

---

## Acceptance Criteria

1. `POST /api/files/upload` with multipart file uploads successfully.
2. Upload validates file type (magic bytes) and returns error for invalid types.
3. Upload validates file size against workspace limit and returns 413 for oversized.
4. Upload returns file metadata: `{ id, key, filename, size, contentType, url }`.
5. `GET /api/files/:fileId` redirects to a signed download URL.
6. `DELETE /api/files/:fileId` deletes the file.
7. `GET /api/files/storage` returns storage usage stats.
8. All routes are authenticated and workspace-scoped.
9. Upload supports both single file and multiple files (array of files).
10. Progress tracking: returns `fileId` immediately, client can poll metadata for processing status.
