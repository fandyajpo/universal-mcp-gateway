Estimated Time: 1 hour

## Goal

Implement the complete password reset flow: request reset email, click link with token, set new password. The reset token is single-use, expires after 1 hour, and is hashed before storage.

---

## Files To Create

- packages/auth/src/services/password-reset-service.ts
- packages/auth/src/emails/password-reset-email.ts
- apps/web/src/app/(auth)/forgot-password/page.tsx
- apps/web/src/app/(auth)/forgot-password/forgot-password-form.tsx
- apps/web/src/app/(auth)/reset-password/page.tsx
- apps/web/src/app/(auth)/reset-password/reset-password-form.tsx
- apps/web/src/actions/auth/password-reset.ts

---

## Acceptance Criteria

1. "Forgot password?" on login page goes to `/forgot-password`.
2. Entering email shows "If an account exists, a reset link has been sent" (no user enumeration).
3. Reset token is single-use, expires in 1 hour.
4. Token hashed before storage (via `@repo/crypto`).
5. Reset password form validates new password requirements.
6. Successful reset invalidates all existing sessions (except current if logged in).
7. Confirmation page: "Password reset successfully — redirecting to login."
