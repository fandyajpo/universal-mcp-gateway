## Goal

Implement email and password authentication using Better Auth's email/password plugin. This covers user registration (with email verification), login, password hashing (via `@repo/crypto`), and account management.

---

## Files To Create

- packages/auth/src/providers/email-password.ts
- packages/auth/src/services/auth-service.ts
- packages/auth/src/emails/verification-email.ts
- packages/auth/src/emails/welcome-email.ts

## Files To Modify

- packages/auth/src/auth-server.ts (add email/password provider)

---

## Requirements

1. User registration with name, email, password. Email verification required before first login.
2. Login validates credentials, creates session, returns session token.
3. Password hashing uses bcrypt cost 12 via `@repo/crypto`.
4. Email verification flow: token sent to user email, verified by clicking link.
5. Rate limiting: 5 login attempts per email per 15 minutes, 3 registration attempts per IP per hour.
6. Account lockout after 10 failed login attempts (30-minute cooldown).
7. Welcome email sent after email verification.

---

## Acceptance Criteria

1. Register creates a user and sends verification email.
2. Verify email with token activates the account.
3. Login with correct credentials returns a session.
4. Login with wrong password increments attempt counter.
5. Account locks after 10 failed attempts.
6. Rate limiting blocks excessive registration attempts.

---

## Notes

For now, "send email" means log the email content to console (development mode) or write to a local file. Production email sending is implemented later. The email service interface in `@repo/auth` abstracts the transport — initial implementation logs to console.
