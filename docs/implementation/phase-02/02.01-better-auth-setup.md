# Step

Step ID: 02.01

Step Name: Better Auth Server Setup

Estimated Time: 1.5 hours

Priority: Critical

Difficulty: Medium

Dependencies: 00.08, 00.03

---

## Goal

Install and configure Better Auth in `@repo/auth` with the MongoDB adapter and all required plugins. Better Auth handles user management, session creation, OAuth, MFA, and password reset out of the box. This step wires Better Auth to the existing database repositories and configures it for the multi-tenant workspace model.

---

## Requirements

1. Create `packages/auth/` with proper `package.json` (depends on `better-auth`, `@repo/database`, `@repo/logger`, `@repo/cache`, `@repo/crypto`, `@repo/validation`).
2. Create `src/auth-server.ts` — Better Auth server instance with:
   - Database adapter: MongoDB via Mongoose schemas (users, sessions, accounts, verifications)
   - Email/password plugin with custom password hashing (using `@repo/crypto`)
   - OAuth plugin (Google, GitHub — configured later)
   - Multi-session plugin (users can have multiple active sessions)
   - Admin plugin (for admin API endpoints)
   - API key plugin (for programmatic access)
   - Magic link plugin (optional, for passwordless login)
3. Create `src/auth-client.ts` — Better Auth client for client-side usage (React hooks).
4. Create `src/plugins/workspace-plugin.ts` — custom Better Auth plugin that:
   - Creates a default workspace on user registration
   - Assigns the `owner` role to the user in that workspace
   - Stores `workspaceId` in the session
5. Configure environment variables: `BETTER_AUTH_SECRET`, `BETTER_AUTH_URL`.
6. Create `src/index.ts` barrel file exporting `auth`, `authClient`, and helper functions.

## Acceptance Criteria

1. Better Auth server initializes without errors.
2. Database tables/schemas are created in MongoDB.
3. `auth.api.createUser()` works through the server.
4. The workspace plugin creates a default workspace on registration.
5. `pnpm typecheck` passes.

---

## Notes

Better Auth requires a secret key: run `pnpm dlx better-auth@latest generate` to create one. The MongoDB adapter uses Mongoose models. The workspace plugin is custom — Better Auth's plugin API allows lifecycle hooks on `onUserCreate`. The auth server is a singleton exported from `@repo/auth` and imported by Next.js Route Handlers.
