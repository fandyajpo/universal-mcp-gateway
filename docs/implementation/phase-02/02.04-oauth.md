# Step

Step ID: 02.04

Step Name: OAuth Providers (Google, GitHub)

Estimated Time: 1.5 hours

Priority: High

Difficulty: Medium

Dependencies: 02.01

---

## Goal

Configure Google and GitHub OAuth providers via Better Auth's OAuth plugin. Users can sign in with their existing Google or GitHub accounts. OAuth tokens are encrypted with `@repo/crypto` before storage. The first OAuth login creates a user account and assigns a default workspace.

---

## Requirements

1. Configure Google OAuth: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` env vars.
2. Configure GitHub OAuth: `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET` env vars.
3. OAuth callback URL: `{APP_URL}/api/auth/callback/{provider}`.
4. On first OAuth login, create user + default workspace + session.
5. OAuth tokens (access, refresh) encrypted with `@repo/crypto` before storage in MongoDB.
6. Link OAuth accounts to existing email-matching user accounts.
7. Handle OAuth errors (user denies, token expired) gracefully with user-facing messages.
8. Add `AUTH_GOOGLE_ID`, `AUTH_GOOGLE_SECRET`, `AUTH_GITHUB_ID`, `AUTH_GITHUB_SECRET` to `.env.example`.

---

## Files To Create

## Files To Modify

- packages/auth/src/auth-server.ts (add OAuth providers)

---

## Acceptance Criteria

1. Login page shows "Continue with Google" and "Continue with GitHub" buttons.
2. Clicking a button redirects to the provider's consent screen.
3. After consent, user is redirected back and authenticated.
4. First-time OAuth user gets a default workspace.
5. Returning OAuth user is logged in and sees their existing workspace.
6. Encrypted tokens are stored in the accounts collection.
7. OAuth error redirects to login page with error message.

---

## Notes

Better Auth's OAuth plugin handles the OAuth flow (authorization URL, callback, token exchange). The encryption layer is added as a custom adapter wrapper. For development, OAuth apps can be created in the Google Cloud Console and GitHub Developer Settings with callback URLs pointing to `http://localhost:3000/api/auth/callback/google` and `http://localhost:3000/api/auth/callback/github`.
