# Step

Step ID: 02.08

Step Name: Login Page

Estimated Time: 1.5 hours

Priority: Critical

Difficulty: Medium

Dependencies: 02.03, 01.01

---

## Goal

Build the login page at `/login` with email/password form, OAuth provider buttons, remember-me toggle, and links to register and password reset pages. The page must handle loading, error, and success states with clear user feedback.

---

## Requirements

1. Form fields: email (required, email validation), password (required), rememberMe (checkbox).
2. "Sign in with Google" and "Sign in with GitHub" buttons with provider icons.
3. "Forgot password?" link.
4. "Don't have an account? Sign up" link.
5. Form validation with Zod (client-side via `@repo/validation`).
6. Error states: invalid credentials, account locked, email not verified, rate limited.
7. Loading state: disabled button with spinner.
8. Success: redirect to `/chat` (or the page the user was trying to access).
9. Remember-me: extends session TTL from 7 days to 30 days.
10. OAuth error handling: show error message from query params (`?error=`).
11. SEO: appropriate meta tags, noindex header.
12. Dark mode support.

---

## Files To Create

- apps/web/src/app/(auth)/login/page.tsx
- apps/web/src/app/(auth)/login/login-form.tsx
- apps/web/src/app/(auth)/login/oauth-buttons.tsx
- apps/web/src/app/(auth)/layout.tsx (auth pages layout, centered card)
- apps/web/src/actions/auth/login.ts (Server Action)

---

## Acceptance Criteria

1. Login form renders with all fields and buttons.
2. Client-side validation shows errors inline.
3. Loading state shows spinner on button.
4. Invalid credentials show error message.
5. Successful login redirects to `/chat`.
6. OAuth buttons redirect to provider consent screen.
7. Remember-me checkbox extends session TTL.
8. Responsive layout works on mobile and desktop.
9. Dark mode renders correctly.

---

## Notes

Auth pages use a shared `(auth)` route group layout that centers the form in a Card component. The login form is a Client Component (`'use client'`) with React Hook Form + Zod resolver. The Server Action `login` calls `@repo/auth`'s `auth.api.signInEmail()` and handles error responses. OAuth redirect is a direct `<a>` tag to `{APP_URL}/api/auth/signin/{provider}`.
