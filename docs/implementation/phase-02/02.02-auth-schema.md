## Goal

Create Zod validation schemas for all authentication-related inputs: login credentials, registration data, password reset requests, OAuth state, MFA verification codes, and session tokens. These schemas are used by the auth API routes to validate incoming requests and provide consistent error messages.

---

## Files To Create

- packages/validation/src/schemas/auth.ts (loginSchema, registerSchema, resetPasswordSchema, verifyEmailSchema, mfaSetupSchema, mfaVerifySchema, oauthSchema, sessionSchema)
- packages/validation/src/index.ts (update barrel exports)

---

## Requirements

1. `loginSchema`: email (valid email, trimmed), password (min 8 chars), rememberMe (optional boolean).
2. `registerSchema`: name (2-100 chars, trimmed), email (valid email), password (min 8, 1 upper, 1 lower, 1 number), confirmPassword (must match password via `.refine()`).
3. `resetPasswordSchema`: token (string min 32), password (same requirements), confirmPassword.
4. `verifyEmailSchema`: token (string min 32).
5. `mfaSetupSchema`: method ("totp"), name (optional display name).
6. `mfaVerifySchema`: code (6 digits), secret (string).
7. `oauthSchema`: provider ("google" | "github"), callbackURL (optional URL).
8. `sessionSchema`: sessionId (string), workspaceId (optional branded ID).

---

## Acceptance Criteria

1. All schemas validate correct input and reject malformed input.
2. `registerSchema` enforces password equality via `.refine()`.
3. Error messages are human-readable.

---

## Notes

Infer TypeScript types from schemas: `type LoginInput = z.infer<typeof loginSchema>`. These types are used in auth service functions. Import base primitives (`emailSchema`, `passwordSchema`) from `@repo/validation` primitives module.
