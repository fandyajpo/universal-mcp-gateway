---

## Goal

Implement multi-factor authentication with TOTP (authenticator app). Users can enroll in MFA from their settings page, scan a QR code with their authenticator app, and verify TOTP codes during login. Includes recovery codes for account access if the authenticator device is lost.

---

## Requirements

1. **Enrollment flow**: User goes to settings → Security → Enable MFA. QR code is displayed (TOTP secret encoded as `otpauth://` URI). User scans with authenticator app and enters a verification code to confirm setup.
2. **Verification during login**: After email/password or OAuth, if MFA is enabled, prompt for TOTP code before session creation.
3. **Recovery codes**: 8 single-use recovery codes generated during enrollment (displayed once, hashed before storage). User can regenerate recovery codes.
4. **TOTP implementation**: Use `otpauth` or `speakeasy` npm package for TOTP generation and verification. 30-second window, 6-digit codes, SHA-1 as specified by RFC 6238.
5. **MFA state**: `mfaEnabled` boolean on user document, `mfaSecret` (encrypted), `recoveryCodes` (hashed with bcrypt).
6. **Disable MFA**: Requires current TOTP code or recovery code to disable.
7. **Remember device**: Optional checkbox "Trust this device for 30 days" — stores a signed token in a cookie, skips MFA prompt on subsequent logins.

## Acceptance Criteria

1. Enrollment generates valid TOTP secret and QR code.
2. Verification code from authenticator app is accepted.
3. Incorrect verification code shows error.
4. MFA prompt appears during login when MFA is enabled.
5. Recovery code can bypass MFA (single-use).
6. Disabling MFA requires current TOTP or recovery code.
7. "Trust this device" skips MFA for 30 days.
8. Recovery codes are displayed once and can be regenerated.

---

## Notes

Use `speakeasy` for TOTP (lightweight, well-tested). QR code rendering: `qrcode` npm package generates a data URL from the `otpauth://` URI, displayed as an `<img>` tag. The MFA secret is encrypted with `@repo/crypto` before storage. Recovery codes are hashed with bcrypt (cost 10, since they're user-supplied at use time).
