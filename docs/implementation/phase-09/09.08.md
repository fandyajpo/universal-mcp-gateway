## Goal

Implement a prompt template system that manages system prompts, user prompts, and few-shot examples with variable interpolation, versioning, and token budget tracking. Templates are stored as configurable strings with `{{variable}}` placeholders and are used by the AI Gateway to construct final prompts before sending to the model.

## Business Context

Prompt engineering is a critical skill for AI application quality. The prompt template system allows prompt changes without code deployments, supports workspace-specific customizations, and ensures consistent system instructions across all AI interactions. It also helps manage token budgets by estimating template token usage before the API call.

## Technical Context

Templates are stored as strings with `{{variable}}` interpolation syntax. Variables can be required or optional with default values. The template system supports conditional sections (`{{#if condition}}...{{/if}}`), iteration (`{{#each list}}...{{/each}}`), and nested templates. Token estimation helps the context builder (Step 08.06) allocate budget accurately.

## Requirements

### Functional
- Template syntax: `{{variable}}` for simple variable interpolation
- Default values: `{{variable:defaultValue}}` for optional variables
- Conditional sections: `{{#if condition}}content{{/if}}` — include content only if condition is truthy
- Iteration: `{{#each list}}Item: {{this}}{{/each}}` — repeat content for each item in array
- Nested templates: `{{> templateName}}` — include another template by name (template composition)
- `render(templateName, variables): string` — render a template with variables
- `renderMessages(templateName, variables): Message[]` — render a template into an array of chat messages (system, user, assistant)
- Template registry: templates stored with name, version, content, description, and tags
- Built-in templates (stored as defaults):
  - `system/default` — default system prompt for chat
  - `system/rag` — system prompt for RAG-grounded chat
  - `system/tools` — system prompt for tool-using chat
  - `user/query` — default user query wrapper
- Template versioning: templates have integer versions; rendering uses latest version unless specified
- Token estimation: `estimateTokens(templateName, variables): number` — estimate rendered token count
- Template validation: validate that all required variables are provided, no unused variables (configurable)
- Template storage: in-memory registry with optional MongoDB persistence for workspace-specific overrides

### Non-functional
- Template rendering: < 5 ms for typical templates
- Token estimation: < 2 ms (fast heuristic, not exact)
- Template compilation: compiled once, cached for reuse

### Performance
- Templates are compiled to render functions on first use (cached in memory)

### Security
- Template content is sanitized — no injection of malicious content via variables
- Variables are escaped for HTML if rendered in UI context (but for LLM prompts, raw values are used)
- Workspace-specific templates are scoped — one workspace cannot access another's templates

- N/A (internal template system)

## Architecture Constraints

- Template syntax is custom (not Handlebars/Mustache) to avoid heavy dependencies — simple regex-based interpolation
- Templates are stored as TypeScript/JSON strings with a registry, not as files on disk (for portability)
- Workspace-specific template overrides are stored in MongoDB and merged with defaults at render time
- The template engine is synchronous (no async operations in rendering)
- Token estimation is a rough heuristic: `text.length / 4` for English, adjustable per character type

## Acceptance Criteria

- `render("system/default", {})` returns the default system prompt
- `render("system/rag", { context: "..." })` includes the context in the rendered prompt
- `render("user/query", { query: "Hello" })` returns a formatted user message
- Variables with defaults work when variable is missing
- `{{#if showContext}}content{{/if}}` conditionally includes content based on variable truthiness
- `{{#each items}}Item: {{this}}{{/each}}` iterates over array variables
- Template versioning: requesting version 1 returns older template version
- Token estimation returns a reasonable estimate
- Missing required variable throws clear validation error
- Template rendering completes in < 5 ms

## Notes

- Template example:
  ```
  You are a helpful AI assistant. Answer the user's question based on the provided context.

  {{#if context}}
  Context:
  {{context}}
  {{/if}}

  {{#if conversationHistory}}
  Previous conversation:
  {{#each conversationHistory}}
  {{role}}: {{content}}
  {{/each}}
  {{/if}}

  User question: {{query}}
  ```
- Template compilation: parse template once, extract variable names and structure, cache compiled function
- For workspace-specific overrides, use template name pattern: `workspace:{workspaceId}:{templateName}`
- Token estimator: `Math.ceil(rendered.length / 4)` — adjust for code or CJK characters
