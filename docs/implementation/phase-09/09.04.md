## Goal

Implement streaming support for the AI Gateway, enabling real-time token-by-token delivery of AI responses to clients. The stream manager handles SSE (Server-Sent Events) parsing from the provider, backpressure management, cancellation, chunk aggregation for tool call detection, and forwarding to the API response stream.

## Business Context

Real-time token streaming is the defining UX feature of modern AI chat interfaces. Users expect to see responses appear token by token, not wait for complete generation. Streaming also enables early detection of tool calls mid-stream and immediate display of citations as they're generated.

## Technical Context

The AI Gateway's Stream Manager receives the AsyncGenerator from the provider's streaming method, processes each chunk for tool call detection (JSON parsing of delta content), manages backpressure (pausing consumption when the client is slow), and forwards chunks to the HTTP response as SSE events. It also handles stream cancellation (client disconnects → abort provider request) and stream errors (partial response on error).

## Requirements

### Functional
- `createStream(provider, request, options)` — create a managed stream
- Accept AsyncGenerator from provider's `chatCompletionStream()`
- Parse SSE chunks from provider: extract delta content, finish reason, tool calls
- Accumulate partial tool call deltas until complete tool call is formed
- Forward typed `StreamEvent` objects to the consumer:
  - `{ type: "token", content: string }` — single token delta
  - `{ type: "tool_call_start", toolCallId, name, arguments: string }` — partial tool call args
  - `{ type: "tool_call_complete", toolCallId, name, arguments: object }` — complete tool call
  - `{ type: "citations", sources: Citation[] }` — citation metadata
  - `{ type: "finish", finishReason: string, usage: TokenUsage }` — stream complete
  - `{ type: "error", error: ProviderError }` — stream error
- Support cancellation: consumer can call `controller.abort()` to cancel the stream
- On abort: signal AbortController to provider → provider stops generation
- Backpressure handling: buffer up to 100 chunks, then pause provider consumption
- Stream timeout: if no chunks received for 60 seconds, abort and emit error
- Support `onToken`, `onToolCall`, `onFinish` callbacks for programmatic consumers
- Forward stream to HTTP SSE response: `data: {json}\n\n` format for each event
- Handle client disconnect: detect `res.socket.destroyed` and abort stream

### Non-functional
- Token forwarding latency: < 10 ms from provider chunk arrival to consumer delivery
- Chunk buffering: never buffer more than 100 chunks
- Backpressure detection: pause provider if consumer buffer exceeds 50 chunks

### Performance
- Zero-copy chunk processing where possible (avoid JSON.parse/stringify cycles)
- Memory: max 100 chunks in buffer

### Security
- Stream content is not logged (token-by-token logging would be massive)
- Only the last 10 chunks are retained for debugging

### Accessibility
- N/A (internal streaming layer, SSE output consumed by frontend)

## Architecture Constraints

- The stream manager is transport-agnostic — it emits typed events; formatting to SSE or WebSocket happens at the API layer
- Tool call accumulation: most providers (OpenAI, Anthropic) emit tool calls as streaming deltas; the accumulator merges deltas until the complete tool call is received
- The stream abort mechanism: `consumer.controller.abort()` → `providerRequestController.abort()` → fetch AbortController triggers cancellation
- Backpressure is cooperative: the consumer signals when it's ready for more chunks; the provider pauses consumption

## Acceptance Criteria

- Stream from provider yields individual token events
- Tool calls in streaming mode are accumulated and emitted as complete events
- Stream cancellation immediately stops provider request
- Client disconnect triggers stream cancellation
- Backpressure pauses provider consumption when buffer exceeds threshold
- Stream timeout (60 seconds idle) emits error event
- SSE formatted stream can be consumed by browser `EventSource` or `fetch` with streaming
- Finish event includes correct finish reason and token usage

## Notes

- SSE format: `data: {"type":"token","content":"Hello"}\n\n`
- Tool call streaming from OpenAI: each chunk may have `delta.tool_calls[0].function.arguments` — append to accumulator
- When a tool call delta is complete (finish_reason: "tool_calls"), emit `tool_call_complete` with parsed JSON arguments
- Backpressure implementation: simple counter-based — increment on push, decrement on pop; when count > 50, pause provider; when < 20, resume
- For HTTP SSE response in Next.js: use `new ReadableStream()` with a controller that pushes SSE-formatted data
