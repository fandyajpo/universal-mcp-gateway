## Goal

Build the marketplace API that provides programmatic access to listing discovery, search, installation, and management. The API supports both public endpoints (browsing, searching, viewing listings) and authenticated endpoints (installation, management).

---

## Business Context

The marketplace API enables automated discovery and installation of connectors. It allows workspace admins to programmatically install, update, and manage connectors without using the UI. It also enables CI/CD pipelines for connector deployment and integration with external tools.

---

## Technical Context

The API follows the same patterns as existing API routes in the platform: Next.js Route Handlers with Zod validation, auth middleware, and service layer delegation. The API supports cursor-based pagination for listing discovery, search with full-text query support, and version-specific installation.

## Requirements

### Functional requirements

- `GET /api/marketplace/listings` — list published listings with pagination, filtering, sorting
- `GET /api/marketplace/listings?search={query}` — full-text search
- `GET /api/marketplace/listings?category={category}` — filter by category
- `GET /api/marketplace/listings?sort={field}&order={asc|desc}` — sorting by name, date, rating, popularity, price
- `GET /api/marketplace/listings/{slug}` — listing detail with full metadata
- `GET /api/marketplace/listings/{slug}/versions` — version history
- `GET /api/marketplace/listings/{slug}/versions/latest` — latest compatible version
- `POST /api/marketplace/listings/{slug}/install` — install connector to workspace (auth required)
- `POST /api/marketplace/listings/{slug}/uninstall` — uninstall connector (auth required)
- `GET /api/marketplace/listings/{slug}/reviews` — reviews for listing
- `GET /api/marketplace/installed` — list installed connectors for workspace (auth required)
- `GET /api/marketplace/updates` — check for updates to installed connectors (auth required)
- `GET /api/marketplace/categories` — list available categories
- `POST /api/marketplace/listings` — create new listing (publisher auth required)
- `PUT /api/marketplace/listings/{slug}` — update listing (publisher auth required)
- `DELETE /api/marketplace/listings/{slug}` — archive listing (publisher auth required)

### Non-functional requirements

- Public endpoints must be rate-limited (60 req/min per IP)
- Authenticated endpoints must use workspace rate limits (200 req/min)
- All responses must use consistent JSON envelope format: `{ data, pagination?, error? }`
- Cursor-based pagination for list endpoints with configurable page size (default 20, max 100)
- Cache-Control headers for public listing data (5-minute TTL)

### Performance requirements

- Listing list: < 200ms
- Listing detail: < 100ms (cached)
- Install/uninstall: < 2s
- Updates check: < 500ms

### Security requirements

- Install/uninstall endpoints require workspace admin permission
- Listing creation/update endpoints require publisher or admin permission
- Public endpoints must not leak draft, rejected, or archived listings
- API keys with `marketplace:read` and `marketplace:write` scopes must be supported

### Accessibility requirements

- API responses must follow standard conventions
- Error messages must be clear and actionable
- Pagination metadata must include total count for UI display

## Package Boundaries

The API routes in `apps/web` delegate to `@repo/marketplace` services. The marketplace package provides serializers, pagination utilities, and error types used by the API layer. No API-specific logic lives in the marketplace package — it provides business logic only.

---

## Architecture Constraints

- All list endpoints use cursor-based pagination with `cursor` (opaque string) and `limit` parameters
- Search uses MongoDB text index with `$text` operator and `$meta: "textScore"` for relevance sorting
- Install endpoint triggers connector registry installation and creates an installation record
- Response envelope: `{ data: T, pagination?: { cursor, limit, total, hasMore }, error?: { code, message, details } }`
- Cache-Control: `public, max-age=300, s-maxage=300, stale-while-revalidate=60` for listing data
