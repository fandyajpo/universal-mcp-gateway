## Goal

Build the marketplace user interface in the web app. Users can browse, search, view details, and install connectors. The UI provides a rich browsing experience with categories, search, filtering, sorting, and detailed listing pages.

---

## Business Context

The marketplace UI is the primary way users discover and install connectors. A well-designed marketplace UI drives connector adoption, helps users find the right tools, and provides a positive developer experience. The UI must be intuitive, responsive, and fast, showcasing connectors effectively.

---

## Technical Context

The marketplace UI is built with Next.js 15, React 19, TailwindCSS, and shadcn/ui components. It consumes the marketplace API (step 15.07) for data and uses TanStack Query for server state management. The UI consists of: a browse page (grid of listing cards), a search page (results list), a category browser, a listing detail page, and an installation management page.

## Requirements

### Functional requirements

- Browse page: grid of featured and recent listing cards with icon, name, short description, rating, install count, pricing
- Search with live results: search bar with autocomplete suggestions, results page with filters and sorting
- Category browser: browse by category with category icons and listing count
- Listing detail page: hero section (icon, name, author, rating, install count), description (rendered markdown), screenshots gallery, version history, reviews section, pricing and install button
- Installation flow: click install → permission review → confirm → install → success state with post-install instructions
- Installation management: view installed connectors, update available indicator, uninstall
- Developer profile page: list of published connectors by developer
- Responsive design: works on desktop, tablet, and mobile
- Loading states: skeleton loaders for cards and detail page
- Empty states: no results found, no listings in category
- Error states: failed to load, retry button

### Non-functional requirements

- Page load must show skeleton immediately, content within 2 seconds
- Search autocomplete must debounce at 300ms
- Listing images must use Next.js Image optimization
- Pagination must be infinite scroll or load-more button
- All pages must be statically rendered where possible (ISR for listing detail)

### Performance requirements

- Browse page initial load: < 2s (including images)
- Search results: < 500ms
- Listing detail: < 1s (ISR with 5-minute revalidation)
- Image load: < 3s for hero images

### Security requirements

- Install button must check authentication and workspace admin permission
- Pricing must be displayed correctly for free vs paid listings
- Paid listing install flow must redirect to checkout or show purchase confirmation

### Accessibility requirements

- All images must have alt text
- Star ratings must be screen-reader accessible
- Search input must have clear label
- Keyboard navigation through listing grid
- Focus management on detail page load

## Package Boundaries

The marketplace UI depends on `@repo/marketplace` (services and types), `@repo/ui` (Card, Button, Badge, Dialog, Input, Select, Rating components), `@repo/auth` (permission checks), and `@repo/api` (API client). No direct database access from UI.

---

## Architecture Constraints

- Listing cards use Next.js `<Image>` component with Cloudflare Image Resizing for optimized icons and screenshots
- The detail page uses Incremental Static Regeneration (ISR) with `revalidate: 300` (5 minutes)
- Install flow requires workspace context — must redirect to workspace-scoped install if not in workspace context
- Installation management is workspace-scoped — shows only connectors installed in the current workspace
- Search autocomplete uses a debounced API call to the search endpoint with `limit=5`
