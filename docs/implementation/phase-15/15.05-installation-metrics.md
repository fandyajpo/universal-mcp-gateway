## Goal

Implement installation metrics tracking for marketplace listings. Collect and display install counts, active installations, uninstall events, and aggregate usage statistics for each connector.

---

## Business Context

Installation metrics are critical for both publishers and marketplace operators. Publishers need to see how many workspaces have installed their connector, how actively it is used, and what the retention rate is. Marketplace operators use metrics to determine featured listings, understand ecosystem health, and calculate revenue share (step 15.06).

---

## Technical Context

Installation events are tracked through the connector registry (step 11.13). When a connector is installed, uninstalled, or updated, an event is emitted. These events are recorded in an `installation_events` collection and aggregated into installation metrics stored on the listing document and in a separate `installation_stats` collection for historical analysis.

## Requirements

### Functional requirements

- Track install events: when a connector is installed via the marketplace
- Track uninstall events: when a connector is removed
- Track update events: when a connector version is updated
- Compute total installs (all-time count) for each listing
- Compute active installations (currently installed, not uninstalled)
- Compute uninstall rate (uninstalls / total installs)
- Track daily/weekly/monthly active installations (DAU/WAU/MAU for connectors)
- Track connector usage: how many tool calls per day per installation
- Display metrics to publishers in their developer dashboard
- Display public metrics: total installs, active installations on listing detail page
- Support metrics time range queries: 7d, 30d, 90d, all-time
- Update listing document with current install count for search sorting

### Non-functional requirements

- Install events must be recorded in real-time (not batched)
- Metrics aggregation must be async (daily cron) to avoid impacting install performance
- Public metrics display is eventually consistent (up to 24h stale for DAU)
- Publisher metrics are near real-time (updated every 5 minutes)

### Performance requirements

- Install event recording: < 50ms
- Publisher metrics query: < 1s
- Public install count display: < 20ms (cached on listing)

### Security requirements

- Install events must be authenticated (which workspace installed)
- Publishers can only see metrics for their own listings
- Admins can see all metrics
- Uninstall events must be verified (cannot fake uninstalls)

### Accessibility requirements

- Metrics charts must provide data in tabular format for screen readers
- Metric cards must use proper semantic HTML

## Architecture Constraints

- Install events are written by the connector registry when `install()` and `uninstall()` are called
- Active installation count = total installs - total uninstalls (not counting re-installs as separate)
- DAU/WAU/MAU are computed from connector tool call activity, not from install/uninstall events
- The listing document's `installCount` field is updated asynchronously via the daily aggregation cron
- Public metrics are aggregated data only — never expose individual workspace installation data
