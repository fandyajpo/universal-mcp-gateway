## Goal

Implement the reviews and ratings system for marketplace listings. Users can rate connectors (1-5 stars), write detailed reviews, mark reviews as helpful, and admins can moderate reviews for quality and appropriateness.

---

## Business Context

Reviews and ratings are critical for the marketplace ecosystem. They help users make informed installation decisions and provide feedback to developers. A well-designed review system with proper moderation ensures high-quality, trustworthy reviews that benefit both consumers and publishers.

---

## Technical Context

Reviews are stored in a `marketplace_reviews` MongoDB collection. Each review links to a listing and a user. The rating is a 1-5 integer. Reviews can optionally include text. Reviews can be flagged as helpful by other users. Admin moderation supports: approve, reject (with reason), flag for review, and delete. The average rating is computed and cached on the listing document.

## Requirements

### Functional requirements

- Users can write reviews for listings they have installed
- Rating: 1-5 stars with half-star increments
- Review text: optional, up to 5000 characters, supports basic markdown (bold, italic, links, lists)
- Users can edit their own review within 30 days of posting
- Users can delete their own review
- Users can mark reviews as helpful (upvote) — one vote per user per review
- Reviews are sorted by: most helpful (default), newest, highest rated, lowest rated
- Average rating and total review count displayed on listing cards and detail page
- Rating distribution histogram (how many 5-star, 4-star, etc.)
- Admin moderation queue: approve, reject (with reason sent to reviewer), flag for manual review
- Auto-moderation: flag reviews with excessive links, offensive language (blocklist), or suspicious patterns
- Verified purchase badge on reviews from users who have installed the connector
- Developer can respond to reviews (one response per review)

### Non-functional requirements

- Average rating must update within 60 seconds of a new review (async update)
- Review list must load in under 200ms with pagination
- Helpful vote must be idempotent (voting twice toggles)
- Moderation actions must be logged to audit trail

### Performance requirements

- Review list load: < 200ms for 50 reviews
- Helpful vote: < 100ms
- Rating update propagation: < 60s

### Security requirements

- Only users who have installed the connector can write a review
- One review per user per listing (no duplicate reviews)
- Review moderation history must be retained for appeals
- Auto-moderation must not store the review content in logs
- Developer responses must be reviewed by admins if the review is under moderation

### Accessibility requirements

- Star rating input must be keyboard operable
- Review text must be readable with screen readers
- Helpful button must announce current state to screen readers

## Architecture Constraints

- Only users with an active installation of the connector can write a review — verified via connector registry installation records
- Average rating is updated asynchronously: write review → emit event → recalculate average → update listing document
- The listing document caches `averageRating` and `totalReviews` for fast display on listing cards
- Auto-moderation rules: maximum 2 links per review, no offensive language (configurable blocklist), minimum 10 characters for text reviews, no repetitive content (flagged as spam)

## Notes

The auto-moderation system is a first-pass filter — it flags suspicious reviews but does not automatically reject them. Flagged reviews go to the admin moderation queue for manual review. Consider expanding the auto-moderation system with ML-based content analysis in a future iteration. The helpful vote system uses a simple toggle: if the user has already voted helpful, clicking again removes the vote. This prevents gaming the system.
