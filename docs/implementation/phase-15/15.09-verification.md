## Goal

Run the complete verification suite for Phase 15: Marketplace. Execute all tests, verify type safety, lint compliance, build integrity, and perform full manual end-to-end testing of the marketplace flows from both publisher and consumer perspectives.

---

## Business Context

The marketplace is a multi-sided platform serving developers (publishers) and users (consumers). Both sides must work correctly for the marketplace to function. Verification must cover the complete publishing flow, installation flow, review system, and revenue sharing. Any failure in these flows erodes trust and adoption.

---

## Technical Context

Verification follows the project's standard pattern. The marketplace has more integration points than previous phases: it integrates with the connector registry (Phase 11), billing system (Phase 14), and storage system (Phase 5). Cross-step and cross-phase integration tests are essential.

## Requirements

### Functional requirements

- Run all unit tests from steps 15.01-15.08 and verify 100% pass rate
- Run all integration tests for marketplace API
- Test complete publisher flow: create listing → upload manifest → submit for review → admin approve → published
- Test complete consumer flow: browse → search → view detail → install → use connector → review → uninstall
- Test version management: create new version → update notification → install update → rollback
- Test review system: write review → edit → vote helpful → admin moderation
- Test installation metrics: install → verify count increments → uninstall → verify count decrements
- Test revenue sharing: verify earnings calculation with mock purchase data
- Verify marketplace API response correctness and pagination
- Verify data isolation: workspace A cannot see workspace B's installations
- Verify TypeScript strict mode with no errors
- Verify lint and build for web app

### Non-functional requirements

- Test suite must complete in under 10 minutes
- Test coverage must exceed 80% on marketplace service code

### Performance requirements

- Unit tests: < 2 minutes
- Integration tests: < 5 minutes
- Build: < 3 minutes

### Security requirements

- Verify that unauthenticated users cannot install connectors
- Verify that non-admin users cannot access admin review queue
- Verify that publishers can only edit their own listings
- Verify that public API does not expose unpublished listings

### Accessibility requirements

- Quick accessibility check of marketplace UI pages
