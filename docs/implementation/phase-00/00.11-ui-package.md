## Goal

Implement `@repo/ui` — a shared component library built on shadcn/ui primitives. This package provides all the reusable UI components (Button, Card, Input, Dialog, Dropdown, Badge, Avatar, Toast, Skeleton, Separator) that the four Next.js applications will use. The package also includes the shadcn/ui configuration, TailwindCSS CSS variables for the design system tokens, and a consistent component API using `class-variance-authority` for variants.

---

## Business Context

A consistent design system is essential for a multi-app platform. The web app, admin app, docs app, and landing page must share the same visual language — colors, typography, spacing, component behavior. shadcn/ui provides the primitives; `@repo/ui` composes them into the platform's design system with consistent APIs, accessibility, and documentation. This prevents UI drift across apps and accelerates development of new features.

---

## Technical Context

The package scaffold exists with `package.json` (depends on `react`, `react-dom`, `class-variance-authority`, `clsx`, `tailwind-merge`, `lucide-react`). The package uses `tailwindcss` but does NOT require Tailwind to be built — the styles come from the consuming app's Tailwind configuration. Components are built using Radix UI primitives (installed as needed by shadcn/ui). The package does not include any application logic — only presentational components.

## Requirements

### Functional requirements

1. **shadcn/ui initialization** — create `components.json` configuration file at package root:
   - `style: "default"` for shadcn/ui style
   - `tailwind.config.js` with CSS variable-based theme
   - `cssVariables: true` for CSS variable support
   - `rsc: true` for React Server Component compatibility

2. **Component: Button** (`src/button.tsx`):
   - Variants: `default`, `destructive`, `outline`, `secondary`, `ghost`, `link`
   - Sizes: `default`, `sm`, `lg`, `icon`
   - Supports `asChild` pattern from Radix Slot
   - Loading state with spinner
   - Full keyboard navigation

3. **Component: Card** (`src/card.tsx`):
   - Sub-components: `Card`, `CardHeader`, `CardTitle`, `CardDescription`, `CardContent`, `CardFooter`
   - Consistent padding and typography

4. **Component: Input** (`src/input.tsx`):
   - Variants: `default`, `error`
   - Sizes: `default`, `sm`, `lg`
   - Error state styling
   - Icon prefix/suffix support

5. **Component: Dialog** (`src/dialog.tsx`):
   - Sub-components: `Dialog`, `DialogTrigger`, `DialogContent`, `DialogHeader`, `DialogFooter`, `DialogTitle`, `DialogDescription`, `DialogClose`
   - Modal behavior with backdrop
   - Keyboard dismissal (Escape)
   - Focus trap
   - Custom animation via Tailwind

6. **Component: Dropdown** (`src/dropdown.tsx`):
   - Sub-components: `DropdownMenu`, `DropdownMenuTrigger`, `DropdownMenuContent`, `DropdownMenuItem`, `DropdownMenuCheckboxItem`, `DropdownMenuRadioItem`, `DropdownMenuSeparator`, `DropdownMenuShortcut`
   - Keyboard navigation with arrow keys

7. **Component: Badge** (`src/badge.tsx`):
   - Variants: `default`, `secondary`, `destructive`, `outline`, `success`, `warning`

8. **Component: Avatar** (`src/avatar.tsx`):
   - Sub-components: `Avatar`, `AvatarImage`, `AvatarFallback`
   - Sizes: `sm`, `default`, `lg`, `xl`
   - Fallback with initials generation

9. **Component: Toast** (`src/toast.tsx`):
   - Sub-components: `Toast`, `ToastProvider`, `ToastViewport`, `ToastTitle`, `ToastDescription`, `ToastAction`, `ToastClose`
   - Variants: `default`, `destructive`, `success`
   - Auto-dismiss with configurable duration
   - Stacking behavior

10. **Component: Skeleton** (`src/skeleton.tsx`):
    - Variants: `text`, `circular`, `rectangular`
    - Animation via Tailwind `animate-pulse`

11. **Component: Separator** (`src/separator.tsx`):
    - Orientation: `horizontal`, `vertical`
    - Decorative (aria-hidden) by default

12. **Utility functions** (`src/utils.ts`):
    - `cn(...inputs)` — class name merging using `clsx` + `tailwind-merge`
    - `focusRing()` — consistent focus ring styles
    - `variantProps(name, variants)` — helper for CVA variant prop generation

### Non-functional requirements

1. All components must support `className` prop for customization.
2. All interactive components must be keyboard accessible.
3. All components must work as React Server Components (no client hooks unless necessary).
4. Components must accept `ref` forwarding.

### Performance requirements

1. Components must be tree-shakeable — importing `Button` must not import `Dialog`.
2. Bundle size must be minimal — only `clsx`, `tailwind-merge`, `class-variance-authority`, `lucide-react` as runtime dependencies.
3. No CSS-in-JS — styles come from Tailwind classes only.

### Security requirements

1. No `dangerouslySetInnerHTML` usage.
2. All user-provided content must be properly escaped by React.
3. No inline event handlers — use proper React patterns.

### Accessibility requirements

1. All interactive components must have proper ARIA attributes.
2. Dialog must implement modal dialog pattern (aria-modal, role="dialog", focus trap).
3. Dropdown must implement menu pattern (role="menu", aria-orientation).
4. Button must have proper role="button" and keyboard handlers.
5. Toast must implement live region for screen reader announcements.
6. Avatar must have `alt` text via `AvatarFallback`.
7. All components must be tested with keyboard-only navigation.

## Package Boundaries

`@repo/ui` exports:
- All component primitives (Button, Card, Input, Dialog, Dropdown, Badge, Avatar, Toast, Skeleton, Separator)
- `cn` utility function
- `focusRing` utility
- Component prop types (inferred from component definitions)

`@repo/ui` does NOT export:
- Raw Radix UI primitives
- Internal helper components
- Layout components (those belong in the app using this package)
- Page-level components

---

## Architecture Constraints

1. Components must be framework-agnostic — no Next.js-specific imports (no `next/link`, `next/image`).
2. All components must use `'use client'` only when they use React hooks (useState, useEffect, useCallback).
3. Server-compatible components (no hooks) must NOT have the `'use client'` directive.
4. Components must use `forwardRef` for ref forwarding.
5. All styling must use Tailwind classes — no inline styles, no CSS modules.

## Notes
