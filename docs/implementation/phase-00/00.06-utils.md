## Goal

Implement `@repo/utils` — a collection of general-purpose utility functions used across all packages and apps. This package provides async helpers, string manipulation, date/time formatting, object utilities (deep merge, clone, pick, omit), URL helpers, retry logic with exponential backoff, and TypeScript type guards. Keeping these utilities centralized prevents duplication and ensures consistent behavior across the codebase.

---

## Business Context

Enterprise applications accumulate ad-hoc utility functions scattered across files and packages. This leads to inconsistent implementations of common operations (e.g., three different deep-merge functions with subtly different behavior). A single `utils` package that is reviewed, tested, and documented eliminates this fragmentation. It also speeds up development — every engineer knows that common operations live in `@repo/utils`.

---

## Technical Context

The package scaffold has `package.json` with a dependency on `@repo/types`. The package is currently empty. The implementation must follow functional programming principles — all functions should be pure, side-effect-free, and well-typed. Each utility category should be in its own file, barrel-exported from `src/index.ts`.

## Requirements

### Functional requirements

1. **Async utilities** (`src/async.ts`):
   - `sleep(ms)` — returns a promise that resolves after `ms` milliseconds
   - `retry<T>(fn, options?)` — retries an async function with exponential backoff, configurable max retries, initial delay, and max delay
   - `timeout<T>(promise, ms)` — wraps a promise with a timeout that rejects if not settled within `ms`
   - `parallel<T>(tasks, concurrency)` — runs async tasks with controlled concurrency
   - `debounce<T extends (...args: any[]) => any>(fn, ms)` — standard debounce
   - `throttle<T extends (...args: any[]) => any>(fn, ms)` — standard throttle
   - `raceWithTimeout<T>(promises, ms)` — races multiple promises with a timeout

2. **String utilities** (`src/string.ts`):
   - `slugify(text)` — converts text to URL-safe slug
   - `truncate(text, maxLength, suffix?)` — truncates text at word boundary
   - `capitalize(text)` — capitalizes first letter
   - `titleCase(text)` — converts to Title Case
   - `camelToKebab(text)` — converts camelCase to kebab-case
   - `kebabToCamel(text)` — converts kebab-case to camelCase
   - `generateId(length?)` — generates a secure random ID using crypto
   - `maskEmail(email)` — masks email for display (u***@example.com)
   - `pluralize(count, singular, plural?)` — simple pluralization

3. **Date utilities** (`src/date.ts`):
   - `formatDate(date, format?)` — formats a date using Intl.DateTimeFormat
   - `formatRelative(date, base?)` — returns relative time string ("2 hours ago")
   - `isExpired(date)` — checks if a date is in the past
   - `addDays(date, days)` — adds days to a date
   - `diffInDays(a, b)` — returns the difference in days between two dates
   - `toISOString(date)` — safe ISO string conversion
   - `now()` — returns current UTC date

4. **Object utilities** (`src/object.ts`):
   - `deepMerge<T, U>(target, source)` — deep merge with type safety
   - `deepClone<T>(obj)` — structured clone (uses `structuredClone` with fallback)
   - `pick<T, K extends keyof T>(obj, keys)` — picks specified keys from an object
   - `omit<T, K extends keyof T>(obj, keys)` — omits specified keys from an object
   - `isPlainObject(value)` — type guard for plain objects
   - `isEmptyObject(obj)` — checks if an object has no own enumerable keys
   - `mapValues<T, R>(obj, fn)` — maps over object values preserving keys

5. **URL utilities** (`src/url.ts`):
   - `buildUrl(base, path, params?)` — builds a URL with query parameters
   - `addQueryParams(url, params)` — adds query params to an existing URL
   - `removeQueryParams(url, keys)` — removes query params by key
   - `isExternalUrl(url)` — checks if a URL is external
   - `getDomain(url)` — extracts domain from URL

6. **Type guard utilities** (`src/guard.ts`):
   - `isString(value)` — type guard for string
   - `isNumber(value)` — type guard for number
   - `isBoolean(value)` — type guard for boolean
   - `isArray(value)` — type guard for array
   - `isObject(value)` — type guard for non-null object
   - `isFunction(value)` — type guard for function
   - `isDefined<T>(value)` — type guard for non-null, non-undefined
   - `isError(value)` — type guard for Error
   - `assertDefined<T>(value, name?)` — asserts value is not null/undefined or throws

### Non-functional requirements

1. All functions must be tree-shakeable — importing one function must not pull in others.
2. All functions must have complete JSDoc documentation.
3. Functions must handle edge cases gracefully (empty strings, null inputs, undefined values).

### Performance requirements

1. `deepMerge` and `deepClone` must handle circular references gracefully.
2. `sleep` must not block the event loop.
3. `retry` must support jitter in backoff to prevent thundering herd.

### Security requirements

1. `generateId` must use `crypto.randomUUID()` or `crypto.getRandomValues()` — never `Math.random()`.
2. `slugify` must strip HTML tags and dangerous characters.
3. `maskEmail` must not reveal more characters than necessary.

### Accessibility requirements

N/A — utils package has no UI.

## Architecture Constraints

1. Must NOT depend on any other `@repo/` package except `@repo/types` (type-only imports for utility types).
2. Must NOT import from `@repo/logger` — utilities must be side-effect-free.
3. Functions must not mutate their inputs — immutability is required.
4. Must use only Node.js built-in APIs and standard web APIs — no third-party utility libraries (no lodash, ramda, etc.).

---

## Acceptance Criteria

1. `retry` retries a failing function the configured number of times and returns the result on success.
2. `deepMerge` correctly merges nested objects without mutating either input.
3. `slugify("Hello World")` returns `"hello-world"`.
4. `generateId()` returns a 21-character URL-safe string by default.
5. `timeout` rejects with a timeout error if the promise takes too long.
6. `parallel` runs at most `concurrency` tasks simultaneously.
7. `assertDefined(null, "value")` throws a `TypeError` with the message "value is required".
8. All functions are tree-shakeable — confirmed by analyzing bundle output.
9. `pnpm typecheck` and `pnpm lint` pass.

## Notes

- Use `crypto.randomUUID()` for `generateId` — available in Node.js 20+ and modern browsers. Fall back to `crypto.getRandomValues` with base64url encoding if needed.
- For `deepMerge`, handle arrays by concatenating (not merging by index). Arrays of primitives should be deduplicated.
- The `retry` function should support options: `{ maxRetries: 3, initialDelay: 1000, maxDelay: 30000, jitter: true, onRetry: (error, attempt) => void }`.
- All date operations should use UTC internally to avoid timezone-dependent bugs.
- The `formatDate` function should accept an `Intl.DateTimeFormatOptions` object for flexibility.
