## Goal

Implement all entity-specific Mongoose models and repositories extending `BaseRepository` and `TenantAwareRepository` from step 00.08. Six repositories must be created: `UserRepository`, `WorkspaceRepository`, `SessionRepository`, `ApiKeyRepository`, `AuditLogRepository`, and `DocumentRepository`. Each repository includes its Mongoose schema, model, repository class, and unit/integration tests.

---

## Business Context

These six repositories cover the foundational data entities that every feature depends on. Users and sessions are needed for authentication. Workspaces are the core multi-tenant unit. API keys enable programmatic access. Audit logs provide the compliance trail. Documents are the foundation of the knowledge base and RAG pipeline. Getting these repositories right — with proper indexing, tenant isolation, and query performance — establishes the pattern for all future entity repositories.

---

## Technical Context

Each repository builds on the `BaseRepository` and `TenantAwareRepository` classes from step 00.08. Each entity has a Mongoose schema (with indexes), a Mongoose model, and a repository class. Schemas are defined in `src/models/` and repositories in `src/repositories/`. Domain types are imported from `@repo/types`.

## Requirements

### Functional requirements

1. **User model and repository** (`src/models/user.ts`, `src/repositories/user.ts`):
   - Fields: `email`, `emailVerified`, `name`, `image`, `passwordHash`, `roles` (map of workspaceId -> role), `isActive`, `lastLoginAt`, `mfaEnabled`, `mfaSecret`, `recoveryCodes`
   - Indexes: `email` (unique), `name` (text index for search), compound `isActive + lastLoginAt`
   - Repository methods: `findByEmail(email)`, `findActive()`, `searchByName(query)`, `setMfaSecret(userId, secret)`, `verifyRecoveryCode(userId, code)`

2. **Workspace model and repository** (`src/models/workspace.ts`, `src/repositories/workspace.ts`):
   - Fields: `name`, `slug`, `avatar`, `description`, `ownerId`, `memberCount`, `plan`, `isActive`, `settings` (flexible object for feature flags, rate limits, branding)
   - Indexes: `slug` (unique), `ownerId`, compound `isActive + plan`
   - Repository methods: `findBySlug(slug)`, `findByOwner(ownerId)`, `searchByNameOrSlug(query)`, `updateSettings(workspaceId, settings)`

3. **Session model and repository** (`src/models/session.ts`, `src/repositories/session.ts`):
   - Fields: `userId`, `workspaceId`, `token`, `expiresAt`, `ipAddress`, `userAgent`, `isValid`, `lastActivityAt`
   - Indexes: `token` (unique), compound `userId + isValid`, `expiresAt` (TTL index for auto-expiry)
   - Repository methods: `findByToken(token)`, `findActiveByUser(userId)`, `invalidateSession(token)`, `invalidateAllUserSessions(userId)`, `cleanupExpired()`

4. **ApiKey model and repository** (`src/models/api-key.ts`, `src/repositories/api-key.ts`):
   - Fields: `workspaceId`, `name`, `keyHash`, `keyPrefix` (first 8 chars for identification), `scopes`, `expiresAt`, `lastUsedAt`, `isActive`, `createdBy`
   - Indexes: `keyHash` (unique), compound `workspaceId + isActive`, `expiresAt` (partial index for non-expired keys)
   - Repository methods: `findByKeyHash(hash)`, `findByWorkspace(workspaceId)`, `rotateKey(keyId, newHash)`, `revokeKey(keyId)`, `recordUsage(keyId)`

5. **AuditLog model and repository** (`src/models/audit-log.ts`, `src/repositories/audit-log.ts`):
   - Fields: `workspaceId`, `userId`, `action`, `entityType`, `entityId`, `changes` (JSON diff), `ipAddress`, `userAgent`, `metadata`
   - Indexes: compound `workspaceId + createdAt`, compound `userId + createdAt`, compound `action + createdAt`
   - Repository methods: `log(action, data)`, `findByWorkspace(workspaceId, filters?)`, `findByUser(userId, filters?)`, `findByEntity(entityType, entityId)`, `search(filters)` with date range, action type, and pagination

6. **Document model and repository** (`src/models/document.ts`, `src/repositories/document.ts`):
   - Fields: `workspaceId`, `title`, `description`, `source` (upload, webhook, connector, api), `sourceUrl`, `contentType`, `fileSize`, `fileKey` (R2 key), `pageCount`, `status` (processing, ready, error), `metadata`, `tags`, `processedAt`, `uploadedBy`
   - Indexes: compound `workspaceId + status`, compound `workspaceId + contentType`, compound `workspaceId + createdAt`, text index on `title + description + tags`
   - Repository methods: `findByWorkspace(workspaceId, filters?)`, `findByStatus(status)`, `search(query, workspaceId)`, `updateStatus(documentId, status)`, `markProcessed(documentId)`, `countByWorkspace(workspaceId)`

### Non-functional requirements

1. Every repository method must use Mongoose lean queries where appropriate for read performance.
2. All indexes must be defined in the schema file — never created manually in the database.
3. AuditLog create must be fire-and-forget — it must not block the main operation.

### Performance requirements

1. User lookup by email must complete in under 5ms (unique index).
2. Session lookup by token must complete in under 5ms (unique index).
3. Audit log queries with date range must use the compound index and complete in under 50ms for typical result sets.
4. Document search with text query must complete in under 100ms.

### Security requirements

1. Password hashes must never be returned in query results — use `.select('-passwordHash')` by default.
2. API key hashes must never be returned — use `.select('-keyHash')` by default.
3. MFA secrets must never be returned — use `.select('-mfaSecret -recoveryCodes')` by default.
4. All repositories must use `TenantAwareRepository` except for `UserRepository` (users are cross-tenant) and `AuditLogRepository` (audit logs are tenant-scoped via workspaceId).

### Accessibility requirements

N/A — database package has no UI.

## Architecture Constraints

1. `UserRepository` must NOT extend `TenantAwareRepository` — users can belong to multiple workspaces.
2. `WorkspaceRepository`, `SessionRepository`, `ApiKeyRepository`, `DocumentRepository` must extend `TenantAwareRepository`.
3. `AuditLogRepository` must extend `TenantAwareRepository` with `workspaceId` as the tenant field.
4. All text indexes must use `{ weights: { title: 10, description: 5, tags: 3 } }` for relevance scoring.
5. All schemas must use the `baseSchema` definition and `timestampsPlugin` from step 00.08.

7. `pnpm typecheck` and `pnpm lint` pass.

## Notes

- Use `typegoose`-style pattern: define the Mongoose schema, then create the model. Alternatively, use the standard Mongoose `Schema` + `model()` pattern — choose whichever is already partially implemented.
- For the `AuditLog` repository, the `log` method should be synchronous from the caller's perspective — use `await` inside but don't propagate the promise to the caller if performance is critical. Consider an in-memory buffer that flushes periodically.
- For `SessionRepository.cleanupExpired()`, create a TTL index on `expiresAt` that automatically removes expired documents. Mongoose supports `expires: 0` in the index option.
- The `Workspace` model's `settings` field should be a flexible `Schema.Types.Mixed` with validation at the service layer, not the database layer.
- For `DocumentRepository.search()`, use MongoDB's `$text` index with `$meta: "textScore"` for relevance-based sorting.
