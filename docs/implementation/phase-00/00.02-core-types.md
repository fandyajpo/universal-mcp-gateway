## Goal

Implement the `@repo/types` package — the zero-dependency foundation of the entire type system. Every domain entity, DTO, API contract, branded identifier, and enum used across all packages and apps must be defined here. This package must have zero runtime dependencies to ensure it remains the stable root of the dependency graph.

---

## Business Context

The types package is the contract layer of the platform. When the AI Gateway, MCP Gateway, RAG Engine, or any other package needs to pass data between boundaries, the shapes of that data are defined here. Getting these types right early prevents cascading refactors later. The branded ID pattern (e.g., `UserId`, `WorkspaceId`) prevents primitive obsession and makes the type system self-documenting.

---

## Technical Context

The package already has a scaffold with `package.json`, `tsconfig.json`, and `src/index.ts`. The existing `src/index.ts` re-exports from `./types/utils`, `./types/branded`, `./models/*`, and `./enums`. These referenced modules do NOT yet exist — they must be created. The package uses `@repo/types` as its name, TypeScript strict mode, and must remain dependency-free beyond `typescript` as a dev dependency.

## Requirements

### Functional requirements

1. Define branded ID types: `UserId`, `WorkspaceId` (aliased to `TenantId`), `DocumentId`, `ConnectorId`, `AgentId`, `SessionId`, `ApiKeyId`. Each must use TypeScript's `brand` pattern with a unique symbol.
2. Define utility types: `DeepPartial<T>`, `Nullable<T>`, `Optional<T>`, `AsyncReturnType<T>`, `Brand<T, B>`.
3. Define auth model types: `User`, `Session`, `Workspace`, `WorkspaceMember`, `WorkspaceRole` (union type: `"owner" | "admin" | "member" | "viewer"`).
4. Define document model types: `Document`, `DocumentChunk`, `DocumentMetadata`, `DocumentSource`.
5. Define connector model types: `Connector`, `ConnectorConfig`, `ConnectorStatus`, `ConnectorType`.
6. Define agent model types: `Agent`, `AgentConfig`, `AgentStatus`, `AgentTool`.
7. Define chat model types: `ChatMessage`, `ChatThread`, `ChatRole`, `ChatConfig`.
8. Define embedding model types: `Embedding`, `EmbeddingConfig`, `EmbeddingModel`.
9. Define RAG model types: `RAGResult`, `RAGConfig`, `RetrievalStrategy`, `ChunkStrategy`.
10. Define MCP model types: `MCPTool`, `MCPRequest`, `MCPResponse`, `MCPError`, `MCPCapability`.
11. Define platform model types: `ApiKey`, `ApiKeyScope`, `AuditLog`, `AuditAction`, `BillingPlan`, `Subscription`, `FeatureFlag`, `Invitation`, `Notification`, `Permission`, `RateLimit`, `Webhook`.
12. Define enums: `WorkspaceRole`, `ConnectorType`, `ConnectorStatus`, `AgentStatus`, `ChatRole`, `RetrievalStrategy`, `ChunkStrategy`, `AuditAction`, `ApiKeyScope`, `FeatureFlag`.

### Non-functional requirements

1. Zero runtime dependencies — only `typescript` in devDependencies.
2. All types use `interface` for object shapes, `type` for unions/intersections/utility types.
3. All enums use `const enum` or plain `as const` objects to avoid runtime overhead where possible.
4. Every exported symbol has JSDoc describing its purpose.

### Performance requirements

1. Types must not emit any runtime JavaScript — only declaration files.

### Security requirements

1. No secrets, passwords, or tokens stored as literals in type definitions.

### Accessibility requirements

N/A — types package has no UI.

## Package Boundaries

- All branded ID types
- All utility types
- All domain model interfaces
- All enums

`@repo/types` does NOT export:
- Internal helper types
- Implementation details or runtime code

---

## Architecture Constraints

1. Zero runtime dependencies. The `package.json` `dependencies` field must be empty after this step.
2. No `any` types anywhere in the codebase. Use `unknown` with type guards.
3. Branded IDs must use the `Brand<T, B>` utility with unique `symbol` declarations to prevent cross-type assignment.
4. Every interface property must be explicitly typed — no inferred `any`.
5. Enums must use `as const` objects with corresponding TypeScript union types for tree-shakeability.

---

## Acceptance Criteria

3. `pnpm --filter @repo/types lint` exits with code 0.
4. All domain models have at least the required fields documented in the model definitions below.
5. `DeepPartial<User>` correctly makes all properties optional recursively.
6. `WorkspaceRole` union type contains exactly `"owner" | "admin" | "member" | "viewer"`.
