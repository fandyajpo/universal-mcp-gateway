## Goal

Implement the Confluence connector that integrates Confluence Cloud with the platform. The connector provides tools for reading pages, spaces, and attachments, searching Confluence content with CQL. It exposes Confluence content as MCP tools and syncs pages into the knowledge base for RAG retrieval.

## Business Context

Confluence is the most widely used enterprise wiki and documentation platform. Integrating Confluence allows AI models to search across company documentation, product specs, and internal knowledge bases. This is often the single most valuable knowledge source for enterprise RAG.

## Technical Context

The Confluence connector uses the Confluence Cloud REST API. Authentication is via OAuth2 (Atlassian 2LO) or Basic Auth with API token. The connector uses `jira.js` (which also supports Confluence API) or raw HTTP for Confluence's specific endpoints.

## Requirements

### Functional
- **Authentication**: OAuth2 (Atlassian 2LO) or Basic Auth with API token
  - Same Atlassian OAuth2 setup as Jira connector
- **Tools**:
  - `confluence_search` — search content using CQL (Confluence Query Language)
  - `confluence_get_page` — get page content in storage format or view
  - `confluence_list_pages` — list pages in a space with pagination
  - `confluence_list_spaces` — list spaces
  - `confluence_get_space` — get space details
  - `confluence_get_page_children` — get child pages (page tree)
  - `confluence_get_attachments` — list attachments for a page
  - `confluence_get_labels` — get labels for a page
- **Content conversion**: Convert Confluence storage format (XHTML) to markdown
  - Parse XHTML to extract text content
  - Handle tables, code blocks, images, links
  - Convert to clean markdown for RAG consumption
- **Sync**: Incremental sync of pages using `lastModified` filter
  - CQL: `lastModified >= "{date}" ORDER BY lastModified`
  - Convert each page to markdown
  - Store as documents in knowledge base
- **Rate limiting**: Confluence Cloud: 500 req/min per integration
- **Error handling**: Map Confluence API errors

### Non-functional
- XHTML to markdown conversion: < 50 ms for typical pages (< 500 lines)
- Page retrieval: < 1 second for pages with body

### Performance
- Pagination: use `limit` + `start` parameters for all list operations
- Body retrieval: use `body.storage` representation for full content
- Batch content fetch: fetch pages in pages of 25

### Security
- API token / OAuth token stored encrypted
- Read-only access — no write operations
- Content scoped to spaces accessible by the authenticated user

### Accessibility
- N/A (connector service)

- Confluence base URL is required during connector setup (different Atlassian domains per customer)

## Notes

- Confluence API base path: `https://{site}.atlassian.net/wiki/api/v2` (v2 API) or `/rest/api` (v1 API)
- Use Confluence v2 API where available (pages, spaces); fall back to v1 for some endpoints
- Page content retrieval: `GET /wiki/api/v2/pages/{id}?body-format=storage`
- XHTML conversion: use `turndown` library for HTML→markdown conversion
- CQL search: `GET /wiki/rest/api/search?cql={cql}&limit={limit}`
- For sync, CQL: `lastModified >= "{lastSyncAt}" AND type = "page" ORDER BY lastModified`
