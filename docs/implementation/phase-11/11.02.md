## Goal

Implement the `BaseConnector` abstract class that provides default implementations for common connector functionality. The base class handles configuration management, authentication state, tool registration, error handling, logging, and health checks. Connector authors extend this class and override specific methods as needed.

## Business Context

Every connector needs the same boilerplate: configuration validation, auth token management, tool registration, error wrapping, and logging. The base class eliminates this boilerplate, letting connector authors focus on the unique integration logic for their specific service.

## Technical Context

`BaseConnector` implements the `Connector` interface and provides sensible defaults. It manages the config and auth state, provides a standard logger scoped to the connector, handles tool registration with the registry, wraps errors in standard error types, and implements a health check. Concrete connectors extend `BaseConnector` and override `initialize()`, `authenticate()`, and `executeTool()`.

## Requirements

### Functional
- Abstract class `BaseConnector<C, A>` implementing `Connector<C, A>`
- Constructor: `constructor(id: string, config: BaseConfig)` where BaseConfig includes `{ name, version, description, author }`
- Default `initialize(config)`:
  - Validate config against `configSchema` (using ajv)
  - Store validated config in `this.config`
  - Set `this.initialized = true`
  - Log initialization success with connector name and version
- Default `authenticate(auth)`:
  - Validate auth data against auth method's schema
  - Store auth result (credentials + expiration) in `this.auth`
  - Return AuthResult with success status
  - If auth method is OAuth2, attempt token refresh if expired
- Default `getTools()`:
  - Return `this.tools` (set by constructor or overridden)
  - Each tool has `executeTool` bound to `this.executeTool.bind(this, toolName)`
- Default `executeTool(name, params, context)`:
  - Throw `MethodNotImplementedError` (connectors must override)
  - Wraps errors in `ConnectorExecutionError`
- Default `shutdown()`:
  - Clear auth state
  - Set `this.initialized = false`
  - Log shutdown
- Default `healthCheck()`:
  - Returns `{ status: "healthy" | "unhealthy", initialized: boolean, authenticated: boolean, lastAuthAt, lastError }`
- Built-in `getLogger(context?)`: returns scoped logger with connector name and context
- Built-in `validateConfig(config)`: validate against configSchema, return validation errors
- Built-in `handleError(error)`: classify and wrap errors (network, auth, rate limit, unknown)

### Non-functional
- Base class overhead: < 1 ms per method call (simple delegation)
- Base class methods are designed for extension — override any method

### Performance
- Configuration validation on initialize
- Auth state kept in memory; serialized to encrypted storage by the registry

### Security
- Config and auth data stored in protected properties (not enumerable)
- Logger redacts sensitive fields (password, token, secret) based on field names
- Error messages from underlying services are sanitized before propagation

### Accessibility
- N/A (internal base class)

## Architecture Constraints

- BaseConnector is abstract — it cannot be instantiated directly
- Config and auth are stored as `protected` fields for subclass access
- All default methods can be overridden by subclasses
- The base class does NOT register tools with the MCP Gateway registry — that's the connector registry's responsibility
- Logger scoping: `logger.child({ connector: this.id, connectorVersion: this.version })`

## Notes

- The BaseConnector uses the following property naming convention for subclass access:
  - `protected config!: C` — validated configuration
  - `protected auth!: AuthResult` — current auth state
  - `protected initialized = false` — initialization flag
  - `protected tools: ToolDefinition[] = []` — tool definitions
- Logger redaction: wrap the logger with a proxy that sanitizes known sensitive field names
- Error handling: catch errors in executeTool → classify → wrap → rethrow
