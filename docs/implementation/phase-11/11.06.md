## Goal

Implement the Slack connector that integrates Slack workspaces with the platform. The connector provides tools for reading messages, channels, and files, searching Slack history, and receiving real-time events via Slack's Events API. It exposes Slack data as MCP tools for AI model consumption.

## Business Context

Slack is the most common enterprise communication platform. Integrating Slack allows AI models to search past conversations, reference decisions made in channels, and access shared files. The connector turns Slack into a knowledge source for the RAG pipeline.

## Technical Context

The Slack connector uses the Slack Web API (`@slack/web-api`) for REST operations and Slack's Events API for real-time event ingestion. Authentication is via OAuth2 with Slack's Bot token scope. The connector implements `BaseConnector` and provides tools for: listing channels, reading messages, searching history, retrieving files, and getting user information.

## Requirements

### Functional
- **Authentication**: OAuth2 with Bot Token scope
  - Required scopes: `channels:history`, `channels:read`, `files:read`, `search:read`, `users:read`, `team:read`
  - Bot token stored encrypted; auto-refresh not supported by Slack (tokens don't expire)
- **Tools**:
  - `slack_list_channels` — list public channels with pagination (returns `[{ id, name, topic, purpose, memberCount, isArchived }]`)
  - `slack_get_channel_history` — get messages from a channel with pagination (supports `limit`, `cursor`, `oldest`, `latest` timestamps)
  - `slack_search_messages` — search across all channels (supports `query`, `sort`, `count`, `page`)
  - `slack_get_thread_replies` — get threaded replies for a message (requires `channel` and `ts`)
  - `slack_get_file` — get file metadata and download URL (requires `fileId`)
  - `slack_get_users` — list workspace users (returns `[{ id, name, realName, email, isBot }]`)
  - `slack_get_user_info` — get user details by ID
- **Webhook events** (via Slack Events API):
  - `message.channels` — new messages in public channels
  - `message.groups` — new messages in private channels (if bot is a member)
  - `file_shared` — files shared in channels
- **Sync**: Supports incremental sync of channel messages (cursor-based using Slack's `cursor` pagination)
- **Rate limiting**: Respect Slack's tier limits (default: 50 requests per minute for most methods)
- **Error handling**: Map Slack API errors to connector error types; handle rate limits with backoff

### Non-functional
- Channel list caching: cache for 60 seconds (channels don't change frequently)
- Message search timeout: 30 seconds for complex queries
- API call overhead: < 100 ms for simple calls (excluding Slack server latency)

### Performance
- Pagination: use Slack's cursor-based pagination for all list operations
- Batch operations: use Slack's `conversations.list` with `limit=200` for efficient channel listing

### Security
- Bot token scoped to least privilege (only required scopes)
- Messages from private channels only accessible if bot is a member
- File downloads respect Slack's access controls

### Accessibility
- N/A (connector service)

## Architecture Constraints

- Slack Bot token never expires — no token refresh logic needed
- Slack's `search.messages` endpoint is only available on paid Slack plans; handle "paid_only" error gracefully
- Rate limit: Slack returns `Retry-After` header on 429 — implement exponential backoff
- Channel history pagination: use Slack's `cursor` field from `response_metadata`
- The connector uses `@slack/web-api` (not raw HTTP) for Slack API calls

## Notes

- Slack SDK: `@slack/web-api` provides `WebClient` which handles pagination and rate limiting
- Client initialization: `new WebClient(botToken)` where `botToken` is the Slack Bot token from OAuth
- For `slack_get_channel_history`, use `client.conversations.history({ channel, limit, cursor, oldest, latest })`
- For `slack_search_messages`, use `client.search.messages({ query, sort, count, page })`
- Slack Events API requires a public HTTPS URL for verification — configure in the connector registry
