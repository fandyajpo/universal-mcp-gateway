## Goal

Implement the Jira connector that integrates Jira Cloud with the platform. The connector provides tools for reading issues, projects, sprints, and searching with JQL. It exposes Jira data as MCP tools and syncs issues into the knowledge base.

## Business Context

Jira is the most widely used issue tracking and project management tool in enterprise. Integrating Jira allows AI models to query project status, search issues, and reference Jira data. This is essential for enterprise customers who manage their work in Jira.

## Technical Context

The Jira connector uses the Jira Cloud REST API (v3). Authentication is via OAuth2 (2LO) or Basic Auth with API token. The connector uses `jira.js` SDK for typed API access. Jira's API has comprehensive filtering via JQL (Jira Query Language).

## Requirements

### Functional
- **Authentication**: OAuth2 (Atlassian 2LO) or Basic Auth with API token
  - Required scopes: `read:jira-work`, `read:jira-user`, `offline_access`
- **Tools**:
  - `jira_search_issues` — search issues using JQL (supports `jql`, `fields`, `maxResults`, `startAt`)
  - `jira_get_issue` — get issue details with all fields and comments
  - `jira_list_projects` — list projects with metadata
  - `jira_get_project` — get project details with components and versions
  - `jira_get_sprints` — list sprints for a board
  - `jira_get_issue_comments` — get comments for an issue
  - `jira_list_boards` — list Scrum/Kanban boards
  - `jira_get_users` — search users by query
  - `jira_get_fields` — list custom fields available
- **JQL support**: Users can write arbitrary JQL queries for advanced filtering
- **Sync**: Incremental sync of issues using `updated` date filter
  - JQL: `updated >= "{lastSyncDate}" ORDER BY updated ASC`
  - Fetch all issue fields and comments
  - Transform to platform document format
- **Rate limiting**: Jira Cloud rate limits: 500 req/min per integration
- **Error handling**: Map Jira API errors; handle rate limits

### Non-functional
- JQL execution: most complex queries complete within 2 seconds
- Issue detail retrieval: < 1 second for issues with fewer than 50 comments

### Performance
- Pagination: use Jira's `startAt` + `maxResults` for issue lists
- Field selection: request only needed fields via `fields` parameter (don't request `*all`)
- Batch operation: request issues in pages of 50

### Security
- API token / OAuth token stored encrypted
- Read-only access — no issue mutations
- JQL injection: validate JQL strings don't contain write operations

### Accessibility
- N/A (connector service)

## Architecture Constraints

- Jira Cloud REST API v3 is the target; Jira Server/Data Center not supported
- JQL injection prevention: strip or escape semicolons, `DELETE`, `UPDATE`, `CREATE` keywords from user-provided JQL
- The Jira base URL is required during connector setup (different Atlassian domains per customer)
- Authentication: OAuth2 (Atlassian 2LO) is preferred; Basic Auth with API token is fallback
- For field selection, default fields: `summary`, `description`, `status`, `assignee`, `priority`, `issuetype`, `created`, `updated`, `comments`

## Notes

- Jira SDK: `jira.js` provides `Version3Client`
- Client initialization: `new Version3Client({ host: baseUrl, authentication: { basic: { email, apiToken } } })`
- For OAuth2, use Atlassian's 2LO flow: `new Version3Client({ host: baseUrl, authentication: { oauth2: { accessToken } } })`
- Issue search: `client.issueSearch.searchForIssuesUsingJql({ jql, fields, maxResults, startAt })`
- JQL example: `project = "PROJ" AND updated >= "2024-01-01" ORDER BY updated ASC`
- For sync, store last sync date and use `updated >= "{date}"` in JQL
