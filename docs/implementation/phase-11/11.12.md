## Goal

Implement the Google Drive connector that integrates Google Drive with the platform. The connector provides tools for listing files, searching content, reading file metadata, and retrieving file content. It exposes Google Drive data as MCP tools and syncs supported document types into the knowledge base.

## Business Context

Google Drive is the most widely used cloud storage and document collaboration platform. Integrating Google Drive allows AI models to search across documents, spreadsheets, and presentations stored in Drive. This is critical for enterprise customers who manage their documents in Google Workspace.

## Technical Context

The Google Drive connector uses the Google Drive API (v3) and Google Docs API for document content. Authentication is via OAuth2 with Google's OAuth 2.0 for server-side applications. The connector uses the `googleapis` npm package for API access.

## Requirements

### Functional
- **Authentication**: OAuth2 with Google
  - Required scopes: `https://www.googleapis.com/auth/drive.readonly`, `https://www.googleapis.com/auth/docs.readonly`
- **Tools**:
  - `gdrive_list_files` — list files with filters (`q` query, `pageSize`, `orderBy`)
  - `gdrive_search_files` — search files by name and full-text content (`q` query string)
  - `gdrive_get_file` — get file metadata (name, mimeType, size, createdTime, modifiedTime, owners)
  - `gdrive_get_file_content` — get file content (text for Docs/Sheets/Slides, limited for other types)
  - `gdrive_list_folders` — list folders in a parent folder
  - `gdrive_get_file_permissions` — get sharing permissions for a file
  - `gdrive_get_revisions` — list file revisions/version history
- **Supported file types for content extraction**:
  - Google Docs → markdown (via Google Docs API)
  - Google Sheets → CSV/text (first sheet)
  - Google Slides → text from slides
  - PDF → return file ID for PDF processing pipeline
  - Plain text → return as-is
  - Microsoft Office formats → note: limited support, return file as-is
- **Sync**: Incremental sync of files using `modifiedTime` filter
  - Query: `modifiedTime > '{lastSyncAt}' and trashed = false`
  - For Google Docs/Sheets/Slides: fetch content and convert to markdown
  - Store as documents in knowledge base
- **Rate limiting**: Google Drive API: 60,000 requests per 100 seconds per project (adjust quotas)
- **Error handling**: Map Google API errors; handle rate limits

### Non-functional
- File listing: paginated at 100 files per page
- Content retrieval for Google Docs: < 2 seconds for typical documents

### Performance
- Pagination: use `nextPageToken` from Drive API for all list operations
- Content conversion: convert Google Docs to markdown using the Docs API
- Batch operations: request files in parallel (max 5 concurrent)

### Security
- OAuth2 token stored encrypted
- Read-only access — no write operations
- Only files accessible by the authenticated user are visible
- Content extraction respects Google Drive access controls

### Accessibility
- N/A (connector service)

## Architecture Constraints

- Google Drive OAuth2 requires both `drive.readonly` and `docs.readonly` scopes
- The connector uses Google Drive API for file listing and metadata, Google Docs API for content
- Google Sheets content extraction: use `sheets.spreadsheets.values.get` for the first sheet
- Google Slides content extraction: use `slides.presentations.get` and extract text from slide elements
- Sync queries filter to supported file types: `mimeType = 'application/vnd.google-apps.document' or mimeType = 'application/vnd.google-apps.spreadsheet' or mimeType = 'application/vnd.google-apps.presentation' or mimeType = 'text/plain'`
- File exports (for non-Google formats) go through the Phase 05/06 storage pipeline

## Notes

- Google APIs initialization:
  ```typescript
  import { google } from "googleapis";
  const auth = new google.auth.OAuth2(clientId, clientSecret, redirectUri);
  auth.setCredentials({ access_token: accessToken, refresh_token: refreshToken });
  const drive = google.drive({ version: "v3", auth });
  const docs = google.docs({ version: "v1", auth });
  ```
- Drive file list: `drive.files.list({ q: query, pageSize, fields: "files(id,name,mimeType,...)", pageToken })`
- Docs content: `docs.documents.get({ documentId })` — returns structured document with body content
- Google Docs export: use `drive.files.export({ fileId, mimeType: "text/markdown" })` for markdown export
- Drive API query syntax: `name contains 'report' and mimeType = 'application/vnd.google-apps.document'`
