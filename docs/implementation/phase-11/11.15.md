## Goal

Verify the complete Connector SDK and all connector implementations end-to-end. Run comprehensive tests covering the SDK interface, base class, OAuth helper, webhook handler, sync engine, all eight connectors, the connector registry, and installation API. Validate that connectors correctly expose tools, handle authentication, sync data, and integrate with the MCP Gateway.

## Business Context

The connector ecosystem is a major value driver of the platform. If connectors fail to authenticate, return incorrect data, or don't sync properly, the entire knowledge base integration is compromised. Verification ensures all connectors meet quality standards before production use.

## Technical Context

Verification spans unit tests, integration tests (against real third-party APIs with test accounts), and end-to-end tests. Each connector is tested independently for tool correctness, authentication, webhook handling, and sync. The SDK components are tested for interface compliance, error handling, and security.

## Architecture Constraints

- Connector integration tests against real APIs are marked with `@integration` and skipped by default
- Mock HTTP responses for unit tests to avoid API calls
- OAuth2 tests use a mock OAuth server for code exchange
- Webhook tests use known test payloads and signatures
- Registry tests use in-memory MongoDB (mongodb-memory-server)
- End-to-end test mocks the MCP Gateway Tool Registry

## Acceptance Criteria

- All unit tests pass (> 85% coverage on SDK, > 70% on connectors)
- Interface compliance: all connectors implement the full Connector interface
- Base class test: default methods work, overrides work in each connector
- OAuth2 test: URL generation, code exchange, token refresh all correct
- Webhook test: signature verification passes for all supported algorithms
- Sync test: incremental sync fetches only new data; full sync fetches all
- Each connector's tools return correctly formatted data (mocked)
- Registry test: full lifecycle (install → configure → auth → tools → uninstall)
- API tests: all endpoints respond with correct status codes and body
- Security tests: credentials encrypted, no data leakage in error messages
