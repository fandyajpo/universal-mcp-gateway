## Goal

Implement the Linear connector that integrates Linear workspaces with the platform. The connector provides tools for reading issues, projects, and comments, and searching Linear data. It exposes Linear data as MCP tools and syncs issues into the knowledge base.

## Business Context

Linear is a modern project management tool used for issue tracking and sprint planning. Integrating Linear allows AI models to query project status, search issues, and reference project documentation. This is critical for teams using Linear as their primary project management tool.

## Technical Context

The Linear connector uses the Linear GraphQL API. Authentication is via OAuth2 with Linear's API key or OAuth token. The connector uses `@linear/sdk` which provides a typed GraphQL client. Linear's API is GraphQL-based, requiring queries instead of REST endpoints.

## Requirements

### Functional
- **Authentication**: OAuth2 or Personal API Key
  - Required scopes: `read`, `issues:read`, `comments:read`
- **Tools**:
  - `linear_list_issues` ŌĆö list issues with filters (`teamId`, `projectId`, `assigneeId`, `status`, `priority`, `search`)
  - `linear_get_issue` ŌĆö get issue details with description, comments, and metadata
  - `linear_list_projects` ŌĆö list projects with status
  - `linear_get_project` ŌĆö get project details with issues
  - `linear_search_issues` ŌĆö search issues by text query
  - `linear_list_teams` ŌĆö list teams
  - `linear_get_user` ŌĆö get user by ID or email
  - `linear_list_workflow_states` ŌĆö list workflow states (statuses) for a team
- **GraphQL queries**: Each tool uses a specific GraphQL query for efficient data fetching
- **Sync**: Incremental sync of issues using `updatedAt` cursor
  - Fetch issues updated since last sync
  - Include issue comments
  - Transform to platform document format
- **Rate limiting**: Linear allows 1000 requests per minute for OAuth apps; 200 req/min for API keys
- **Error handling**: Map Linear API errors; handle rate limits

### Non-functional
- GraphQL query optimization: request only the fields needed for each tool
- Issue detail query: fetch issue + comments in a single GraphQL query (not two)

### Performance
- Batch issue fetching: use `issues` query with `first: 50` for efficient pagination
- Pagination: use Linear's `after` cursor for all list operations

### Security
- OAuth token stored encrypted
- Read-only access ŌĆö no write operations
- Linear API tokens have scoped permissions

### Accessibility
- N/A (connector service)

## Architecture Constraints

- Linear uses GraphQL API ŌĆö all operations are GraphQL queries, not REST
- The `@linear/sdk` client provides typed methods but can also execute raw GraphQL queries
- Use raw GraphQL queries for tool-specific field selection to minimize data transfer
- Pagination uses `first` + `after` cursor pattern (Relay-style pagination)
- Sync uses `updatedAt` filter: `issues(updatedAt: { after: lastSyncAt })`

## Notes

- Linear SDK: `@linear/sdk` provides `LinearClient` with typed methods
- Client initialization: `new LinearClient({ accessToken })`
- For raw GraphQL queries: `client.client.rawRequest(query, variables)`
- Example issue query:
  ```graphql
  query($id: String!) {
    issue(id: $id) {
      id title description state { name type }
      assignee { id name email }
      team { id name }
      comments { nodes { id body user { name } createdAt } }
    }
  }
  ```
- Sync query: `issues(filter: { updatedAt: { after: { date: "2024-01-01" } } }, first: 50) { nodes { ... } pageInfo { hasNextPage endCursor } }`
