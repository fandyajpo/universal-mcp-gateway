## Goal

Implement a retrieval evaluation framework that measures the quality of the RAG pipeline using standard information retrieval metrics. The framework computes hit rate (recall@K), Mean Reciprocal Rank (MRR), Normalized Discounted Cumulative Gain (NDCG), and precision@K against a ground-truth dataset of query-relevant chunk pairs. Evaluation results guide tuning of chunking, embedding, retrieval, and re-ranking parameters.

## Business Context

Without measurement, it's impossible to know whether the RAG pipeline is improving or degrading. The evaluation framework provides quantitative metrics that inform decisions about chunk size, overlap, retrieval strategy, hybrid weights, and re-ranker selection. It also catches regressions when pipeline components are modified.

## Technical Context

The evaluation framework uses a test dataset of (query, relevant_chunk_ids) pairs. For each query, the framework runs the full RAG pipeline and compares returned results against the ground truth. It computes standard IR metrics and outputs them in a structured report. The framework is designed to be run both offline (during development) and as part of CI (to catch regressions).

## Requirements

### Functional
- Create evaluation test dataset: `[{ query: string, relevantChunkIds: string[], metadata?: object }]`
- Run RAG pipeline (all steps) for each test query
- Compare returned chunks against ground truth
- Compute metrics:
  - `hitRate@K`: whether at least one relevant chunk is in top-K results (K = 1, 3, 5, 10, 20)
  - `mrr@K`: Mean Reciprocal Rank — average of 1/rank of first relevant result
  - `ndcg@K`: Normalized Discounted Cumulative Gain — position-aware ranking quality
  - `precision@K`: fraction of top-K results that are relevant
  - `recall@K`: fraction of all relevant chunks that appear in top-K results
- Support evaluation with different configurations: strategy (vector/hybrid), with/without re-ranker, different topK values
- Generate evaluation report: `{ metrics: { hitRate, mrr, ndcg, precision, recall }, config: { strategy, rerank, topK, topN }, queryCount: number, totalDurationMs }`
- Support running evaluation on a subset of queries (random sample for speed)
- Export results as JSON and formatted table (console output)
- Store evaluation results in MongoDB for tracking over time

### Non-functional
- Evaluation on 100 queries completes in < 5 minutes
- Metrics computation overhead: < 100 ms for 100 queries

### Performance
- Queries run sequentially (not parallel) to avoid rate limit issues
- Each query runs the full RAG pipeline (includes embedding, retrieval, re-ranking)

### Security
- Evaluation dataset does not contain sensitive data
- Test queries and ground truth stored in development/test MongoDB only

### Accessibility
- N/A (developer tooling)

## Notes

- Hit Rate@K: `isHit = groundTruth.some(id => results.slice(0, K).some(r => r.chunkId === id))`
- MRR@K: `rr = 1 / (firstRelevantRank + 1)` if found, else 0; MRR = mean of rr across all queries
- NDCG@K: DCG = sum(rel_i / log2(i+1)), IDCG = DCG of perfect ranking, NDCG = DCG / IDCG
- For simplicity, relevance is binary (0 or 1) — graded relevance can be added later
- Create a ground truth dataset from existing workspace documents by having human annotators tag relevant chunks for test queries
- For CI, use a smaller dataset (20 queries) to keep evaluation time under 1 minute
