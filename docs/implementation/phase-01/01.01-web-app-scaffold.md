## Goal

Scaffold the `apps/web` Next.js 15 application with the App Router, `src/` directory convention, and all necessary configuration files. This is the main user-facing workspace application where users will chat with AI, manage knowledge, and interact with MCP tools. The scaffold must be production-ready with proper TypeScript, ESLint, and build configurations.

---

## Business Context

The web app is the primary interface of the Universal MCP Gateway. Users spend most of their time here — chatting with AI assistants, managing documents, configuring connectors, and collaborating with team members. Getting the scaffold right (standards-compliant, well-structured, extensible) prevents painful restructuring later as features are added.

---

## Technical Context

Next.js 15 with App Router, React 19, TypeScript strict mode, `src/` directory convention. The app depends on `@repo/types`, `@repo/ui`, `@repo/config`, `@repo/utils`, `@repo/validation`, and `@repo/logger`. The scaffold includes `next.config.ts`, `tsconfig.json`, the app directory with root layout, a test page, and environment variable integration.

## Requirements

### Functional requirements

1. Initialize `apps/web` with Next.js 15 App Router and `src/` directory.
2. Create `src/app/layout.tsx` with root HTML, font loading (Inter), metadata, and global styles.
3. Create `src/app/page.tsx` with a placeholder homepage (redirects to `/chat` or shows a welcome page).
4. Create `src/app/chat/page.tsx` placeholder for the main chat interface (empty page shell).
5. Create `src/app/settings/page.tsx` placeholder for settings pages.
6. Configure `next.config.ts` with proper image domains, headers, redirects, and experimental features.
7. Configure `tsconfig.json` with path aliases (`@/` -> `src/`).
8. Create `src/lib/env.ts` that wraps `@repo/config` for Next.js client-side access.
9. Add `postcss.config.mjs` for TailwindCSS processing.
10. Add root `loading.tsx` and `error.tsx` boundary files.

### Non-functional requirements

1. App must use `src/` directory (not pages at root).
2. All pages use Server Components by default — `'use client'` only when needed.
3. Build must produce optimized production output.

### Performance requirements

1. Initial page load must be under 100KB (gzipped) for the landing page.
2. Route segments must be lazy-loaded via Next.js dynamic imports where appropriate.

### Security requirements

1. `next.config.ts` must set security headers (CSP, X-Frame-Options, X-Content-Type-Options).
2. Metadata must not leak internal information.

### Accessibility requirements

1. Root layout must have `lang="en"` attribute.
2. Skip-to-content link must be the first focusable element.
