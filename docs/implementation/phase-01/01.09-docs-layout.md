# Step

Step ID: 01.09

Step Name: Docs Layout and Navigation

Estimated Time: 1 hour

Priority: High

Difficulty: Medium

Dependencies: 01.03, 01.05, 01.06

---

## Goal

Build the documentation site layout with a two-column design: a collapsible sidebar with the doc navigation tree and a main content area. The sidebar shows the documentation hierarchy (Getting Started, Architecture, API Reference, Connectors, Deployment, Contributing). The layout includes a table of contents sidebar on the right for large pages.

---

## Files To Create

- apps/docs/src/components/layout/sidebar.tsx
- apps/docs/src/components/layout/table-of-contents.tsx
- apps/docs/src/components/layout/search.tsx
- apps/docs/src/lib/table-of-contents.ts (extract headings from markdown)
- apps/docs/src/lib/navigation.ts (nav tree structure)

## Files To Modify

- apps/docs/src/app/docs/layout.tsx (integrates sidebar + TOC)
- apps/docs/src/app/layout.tsx (root layout with header/footer)

---

## Acceptance Criteria

1. Docs sidebar renders navigation tree with expandable sections.
2. Table of contents sidebar extracts headings from markdown content.
3. Active page highlighted in sidebar.
4. Active heading highlighted in TOC as user scrolls.
5. Mobile responsive.
6. Build succeeds.

---

## Notes

The navigation tree is defined as a static TypeScript object in `src/lib/navigation.ts`. Each node has: `title`, `href`, `icon?`, `children?`. The sidebar recursively renders the tree. The table of contents is extracted from the rendered markdown's heading structure and highlighted as the user scrolls using IntersectionObserver.
