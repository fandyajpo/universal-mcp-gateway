# Step

Step ID: 01.05

Step Name: TailwindCSS Theme Configuration

Estimated Time: 1 hour

Priority: Critical

Difficulty: Easy

Dependencies: 01.01

---

## Goal

Create a shared TailwindCSS v4 theme configuration with CSS variable-based design tokens, dark mode support, and consistent spacing/typography/color scales across all four apps. The theme configuration lives in `@repo/ui` and is consumed by every app.

---

## Requirements

1. Create `packages/ui/tailwind.config.ts` with:
   - CSS variable-based color tokens (light and dark themes)
   - Custom font family (Inter as primary, mono as secondary)
   - Extended spacing scale
   - Custom breakpoints
   - Animation keyframes (fade-in, slide-up, skeleton pulse)
   - Plugin imports (`tailwindcss-animate`)
2. Define CSS variables in `packages/ui/src/globals.css`:
   - Surface colors: `--background`, `--foreground`, `--card`, `--card-foreground`
   - Brand colors: `--primary`, `--primary-foreground`, `--secondary`, `--secondary-foreground`
   - State colors: `--destructive`, `--warning`, `--success`, `--info`
   - UI colors: `--muted`, `--muted-foreground`, `--accent`, `--accent-foreground`
   - Border/input colors: `--border`, `--input`, `--ring`
   - Radius: `--radius`
   - Dark mode: `.dark` class variant with inverted color values
3. Each app's `tailwind.config.ts` extends the shared config via `presets: [sharedConfig]`.
4. Each app's `globals.css` imports `@repo/ui/globals.css` via `@import "tailwindcss"`.
5. Verify dark mode toggle works: add a `ThemeToggle` component that toggles the `.dark` class on `<html>`.

## Acceptance Criteria

1. All apps render with the same color scheme and typography.
2. Toggling dark mode switches all colors correctly.
3. CSS variables are properly scoped and don't leak.
4. `pnpm build` succeeds for all apps.

---

## Notes

Use `tailwindcss` v4 with CSS-first configuration. The `@repo/ui` package exports the Tailwind config as `preset` for apps to extend. Dark mode uses the `class` strategy (not media query). The `ThemeToggle` component uses Zustand for persistence (zustand store added as a dependency to the web app).
