## Goal

Scaffold the `apps/admin` Next.js 15 application — the organization administration dashboard. This app provides tenant management, user administration, billing oversight, audit log viewing, and system health monitoring. It follows the same conventions as the web app (App Router, src/, shared packages) but has its own layout and route structure.

---

## Requirements

Same scaffold structure as step 01.01 but for the admin app. Routes: `/dashboard`, `/users`, `/workspaces`, `/billing`, `/audit-logs`, `/settings`. The admin app runs on port 3001 by default. It must include the root layout with admin-specific metadata and security headers.

## Notes

The admin app is gated by authentication (Phase 02). For now, all pages are public placeholders. The auth guard middleware will be added in Phase 13. Port is configured in `next.config.ts` via `process.env.ADMIN_PORT || 3001`.
