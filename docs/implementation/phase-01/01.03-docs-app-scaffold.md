## Goal

Scaffold the `apps/docs` Next.js 15 application — the developer documentation and API reference site. This app renders markdown documentation with MDX support for interactive examples. It runs on port 3002.

---

## Requirements

Same scaffold structure as step 01.01 but for the docs app. Key differences: MDX support via `@next/mdx`, documentation route structure (`/docs/getting-started`, `/docs/api`, `/docs/connectors`), and a two-column layout (sidebar navigation + content area). The `next.config.ts` must be configured with MDX support.

---

## Files To Create / Modify

- apps/docs/next.config.ts
- apps/docs/tsconfig.json
- apps/docs/postcss.config.mjs
- apps/docs/mdx-components.ts
- apps/docs/src/app/layout.tsx
- apps/docs/src/app/page.tsx (redirect to /docs)
- apps/docs/src/app/docs/page.tsx
- apps/docs/src/app/docs/layout.tsx
- apps/docs/src/app/docs/[[...slug]]/page.tsx (catch-all for doc pages)
- apps/docs/src/app/loading.tsx
- apps/docs/src/app/error.tsx
- apps/docs/src/app/not-found.tsx
- apps/docs/package.json

---

## Acceptance Criteria

1. `pnpm dev --filter @repo/docs` starts on port 3002.
2. `/docs` route renders with placeholder content.
3. MDX files render correctly (test with a sample `.mdx` file).
4. Build succeeds.

---

## Notes

Add `@next/mdx` and `@mdx-js/loader` as dependencies. Create `apps/docs/content/` directory for markdown documentation files. The catch-all route `[[...slug]]` renders the appropriate markdown file from the content directory. The `mdx-components.ts` file registers custom components for MDX rendering.
