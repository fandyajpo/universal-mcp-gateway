## Goal

Build the complete layout shell for the web app: sidebar navigation, top bar with user menu and workspace switcher, main content area, and a right panel for context/threads. The layout must be responsive (collapsible sidebar on mobile), support dark mode, and include placeholder components for features to be built in later phases.

---

## Requirements

1. **Sidebar** (`src/components/layout/sidebar.tsx`):
   - Logo/wordmark at top
   - Primary nav: Chat, Knowledge Base, Connectors, Agents, Documents
   - Secondary nav: Settings, Help
   - Collapsible (icon-only on mobile, full on desktop)
   - Active route highlighting
   - Keyboard shortcut: `Cmd+B` to toggle
   - `max-w-[280px]` when expanded, `w-[60px]` when collapsed

2. **Top bar** (`src/components/layout/topbar.tsx`):
   - Workspace switcher dropdown (left side, placeholder)
   - Search bar (middle, placeholder)
   - User menu (right side): avatar, name, workspace settings, logout
   - Theme toggle
   - Notification bell (placeholder)

3. **Main content area** (`src/app/layout.tsx`):
   - Sidebar | TopBar + Content
   - Responsive: mobile stacks sidebar as overlay drawer (Sheet component)
   - Breadcrumbs below top bar (auto-generated from route)

4. **Right panel** (`src/components/layout/context-panel.tsx`):
   - Context/thread sidebar (right side, togglable)
   - Shown by default on chat page
   - Hideable via `Cmd+I` shortcut

5. **State management** (`src/lib/store/layout.ts`):
   - Zustand store for sidebar state (collapsed/expanded), right panel visibility
   - Persisted to localStorage

## Acceptance Criteria

1. Sidebar renders with all navigation items, collapses/expands correctly.
2. Top bar shows user menu placeholder, theme toggle works.
3. Right panel toggles open/closed.
4. Layout is responsive — mobile sidebar uses overlay drawer.
5. Keyboard shortcuts work (Cmd+B for sidebar, Cmd+I for right panel).
6. Active route is highlighted in sidebar.
7. `pnpm build` succeeds.

## Notes

Use `lucide-react` for all icons (MessageSquare, BookOpen, Puzzle, Bot, FileText, Settings, HelpCircle, Search, Bell, ChevronDown, PanelRight, Menu, X). The sidebar uses `next/navigation` `usePathname()` for active route detection. The Zustand store for layout is client-only (`'use client'` boundary in provider component that wraps the app).
