---

## Goal

Scaffold the `apps/landing` Next.js 15 application — the marketing landing page. This app is publicly accessible, does NOT require authentication, and is optimized for static export (output: `export`). It runs on port 3003.

---

## Requirements

Same scaffold structure as step 01.01 but for the landing app. Key differences: `output: "export"` in `next.config.ts` for static hosting, no dynamic routes (SSG only), single-page layout with sections (hero, features, pricing, CTA, footer), and lightweight bundle (minimal JavaScript). The landing app does NOT use `@repo/auth` or any database packages.

---

## Files To Create / Modify

- apps/landing/next.config.ts
- apps/landing/tsconfig.json
- apps/landing/postcss.config.mjs
- apps/landing/src/app/layout.tsx
- apps/landing/src/app/page.tsx (landing page with sections)
- apps/landing/src/app/loading.tsx
- apps/landing/src/app/not-found.tsx
- apps/landing/package.json

---

## Acceptance Criteria

1. `pnpm dev --filter @repo/landing` starts on port 3003.
2. Landing page renders with all sections visible.
3. `pnpm build --filter @repo/landing` produces a static export in `out/`.
4. TypeScript compilation passes.

---

## Notes

Static export means no API routes, no server actions, no `getServerSideProps`. All content is compiled at build time. The landing app uses `next/image` with `unoptimized` for static export compatibility. SEO metadata (Open Graph, Twitter cards, structured data) should be configured in the root layout.
