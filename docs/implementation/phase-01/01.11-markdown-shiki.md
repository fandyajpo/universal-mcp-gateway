Estimated Time: 1 hour

---

## Goal

Integrate `react-markdown` with Shiki syntax highlighting for the docs app. Documentation pages authored in Markdown/MDX must render with beautifully highlighted code blocks supporting all major languages (TypeScript, JavaScript, Python, Bash, JSON, YAML, SQL, etc.). The integration must support copy-to-clipboard on code blocks and line highlighting.

---

## Requirements

1. Install `react-markdown`, `remark-gfm`, `rehype-raw`, `shiki`, `@shikijs/rehype` (or `rehype-highlight`).
2. Create `src/components/markdown.tsx` — a Server Component that renders markdown content with Shiki highlighting.
3. Create `src/components/code-block.tsx` — a Client Component for interactive code blocks (copy button, line highlighting, language label).
4. Configure Shiki theme: `one-dark-pro` for dark mode, `one-light` for light mode, switch based on `prefers-color-scheme`.
5. Support custom markdown components: `Alert`, `Tabs`, `Steps`, `Callout`, `Kbd`, `Image` (optimized with next/image).
6. Create `src/app/docs/[[...slug]]/page.tsx` catch-all route that reads markdown files from `content/` and renders them.
7. Frontmatter parsing: `gray-matter` for extracting title, description, order from markdown files.

---

## Acceptance Criteria

1. Markdown content renders with proper headings, lists, tables, and links.
2. Code blocks are highlighted with Shiki — syntax coloring is correct.
3. Copy button copies code block content to clipboard.
4. Language label displayed on code blocks.
5. Line highlighting works (`// highlight-next-line` or `[!code highlight]`).
6. Dark/light theme switching works on code blocks.
7. Frontmatter metadata (title, description) is extracted and used for page title/SEO.
8. Build succeeds.

---

## Notes

The catch-all route `[[...slug]]` reads markdown files from `content/` directory. The slug path maps to the file system path: `content/getting-started.mdx` → `/docs/getting-started`. Frontmatter is parsed with `gray-matter` and used for page metadata. Shiki themes are loaded at build time (not client-side) for zero runtime overhead. The `code-block` component uses `'use client'` for the copy-to-clipboard interaction.
