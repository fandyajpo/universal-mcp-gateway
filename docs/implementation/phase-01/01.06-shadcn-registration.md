# Step

Step ID: 01.06

Step Name: shadcn/ui Component Registration

Estimated Time: 1 hour

Priority: High

Difficulty: Easy

Dependencies: 01.01, 00.11

---

## Goal

Register and customize shadcn/ui components in each app. The `@repo/ui` package provides the primitives; each app creates a `components/ui/` re-export layer with app-specific customizations (theming, defaults, additional variants). This ensures the components use each app's TailwindCSS theme correctly.

---

## Requirements

1. Each app creates a `src/components/ui/` directory that re-exports from `@repo/ui` with app-specific configurations.
2. The `components.json` in each app points to the correct Tailwind config and CSS file.
3. Register at minimum: Button, Card, Input, Dialog, Dropdown, Badge, Avatar, Toast, Skeleton, Separator, Label, Switch, Tabs, Command (for combobox), Sheet (for slide-out panels), Tooltip.
4. Add app-specific CSS overrides in each app's `globals.css` for any component that needs customization.
5. Create a test page in each app that renders all registered components for visual verification.

---

## Files To Create / Modify

- apps/web/components.json
- apps/web/src/components/ui/index.ts (re-exports from @repo/ui with app defaults)
- apps/web/src/app/_examples/components/page.tsx (test page)
- apps/admin/components.json
- apps/admin/src/components/ui/index.ts
- apps/admin/src/app/_examples/components/page.tsx
- apps/docs/components.json
- apps/docs/src/components/ui/index.ts
- apps/landing/components.json
- apps/landing/src/components/ui/index.ts

---

## Acceptance Criteria

1. All registered components render correctly in each app.
2. App-specific customizations (e.g., button border radius) are applied.
3. Components are tree-shaken — unused components don't appear in the bundle.
4. Test pages show all components with correct styling.

---

## Notes

The re-export layer is thin: each file imports from `@repo/ui/button` and re-exports with app-specific `cva` variants or default props. The test page (`/examples/components`) is removed before production deployment. Use `npx shadcn@latest add` to add components to each app, then move them to `@repo/ui`.
