---

## Goal

Build the admin dashboard layout: fixed sidebar with navigation groups, top bar with admin-specific actions, and content area. The admin layout is distinct from the web app layout — it uses a dark sidebar by default, shows system-wide data, and includes navigation groups for users, workspaces, billing, audit logs, and settings.

---

## Requirements

1. **Sidebar** — dark-themed, non-collapsible on desktop, overlay drawer on mobile. Sections: Overview, Management (Users, Workspaces, Connectors), Operations (Audit Logs, Queue, Health), Billing (Overview, Plans, Invoices), Configuration (Settings, Feature Flags, Announcements).
2. **Top bar** — admin badge, search, notifications, support ticket count, user menu.
3. **Content area** — full-width below sidebar with padding.
4. **Breadcrumbs** — auto-generated from route path.
5. **State** — Zustand store for mobile sidebar.

---

## Notes

Admin layout is visually distinct: sidebar uses `bg-slate-900` with white text, top bar uses `bg-background` with border-bottom. The admin badge is a small indicator in the top bar showing "Admin" with a Shield icon. All components are Server Components where possible — only the mobile sidebar toggle needs client interactivity.
