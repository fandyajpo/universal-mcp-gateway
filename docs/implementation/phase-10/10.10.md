## Goal

Verify the complete MCP Gateway end-to-end. Run comprehensive tests covering all components: protocol handler, tool registry, validation, sandbox, streaming, versioning, error handling, discovery API, and API routes. Validate that the gateway correctly implements the MCP specification, executes tools safely, and integrates with clients.

## Business Context

The MCP Gateway is the tool execution infrastructure for the entire platform. If it fails, connectors (Phase 11) and chat tool integration (Phase 12) cannot function. Verification ensures MCP specification compliance, sandbox security, and reliable tool execution.

## Technical Context

Verification spans unit tests, integration tests (with MCP Inspector or test client), and end-to-end tests. Test scenarios cover the full MCP session lifecycle, all error conditions, sandbox enforcement, version resolution, and API route behavior.

## Requirements

### Functional
- Verify protocol handler correctly implements MCP specification (initialize, ping, tools/list, tools/call)
- Verify tool registry CRUD with MongoDB persistence
- Verify JSON Schema validation rejects invalid params
- Verify sandbox enforces timeout, memory, output limits, and network restrictions
- Verify streaming tool responses emit progress notifications
- Verify tool versioning resolves semver ranges correctly
- Verify error handling returns correct JSON-RPC error codes for all failure modes
- Verify discovery API returns permission-filtered tool listings
- Verify API routes (SSE, message, REST CRUD) respond correctly
- Verify end-to-end: MCP client connects → initializes → lists tools → calls tool → receives result

### Non-functional
- Test suite completes within 10 minutes
- Integration tests use real MongoDB and worker threads
- Test coverage: > 85% on core MCP logic

### Performance
- Protocol overhead: < 10 ms per message
- Tool lookup: < 5 ms (cached)
- Validation: < 5 ms per call
- SSE connection setup: < 100 ms
- Worker pool reuse: execution reuses worker without cold start

### Security
- Sandbox isolation: tool cannot access parent process resources
- Network restriction: non-allowed endpoints blocked
- Error messages: no internal details leaked
- Auth enforcement: unauthenticated requests return 401

### Accessibility
- N/A (verification tests)
