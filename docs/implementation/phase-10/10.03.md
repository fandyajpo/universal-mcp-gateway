## Goal

Implement input validation for tool calls using JSON Schema validation. When a tool call request arrives (via `tools/call`), the validator checks that the provided parameters conform to the tool's input schema before execution begins. Invalid parameters are rejected with clear error messages identifying which fields failed validation.

## Business Context

AI models sometimes generate invalid tool call parameters — missing required fields, wrong types, or values outside allowed ranges. Validating parameters before execution prevents cryptic runtime errors and ensures tools receive well-formed inputs. Clear validation error messages also help the AI model correct its next attempt.

## Technical Context

JSON Schema validation is performed using the `ajv` library (the fastest JSON Schema validator for Node.js). Each tool's `inputSchema` is compiled into a validation function on registration and cached for reuse. Validation errors are mapped to human-readable messages with JSON Pointer paths to the invalid field.

## Requirements

### Functional
- `validateToolParams(toolDefinition, params): ValidationResult` — validate parameters against tool's input schema
- Validate required fields are present
- Validate field types match schema (string, number, object, array, boolean)
- Validate string patterns, minLength, maxLength
- Validate number ranges (minimum, maximum, exclusiveMinimum, exclusiveMaximum)
- Validate array minItems, maxItems, uniqueItems
- Validate enum values
- Validate nested object structures recursively
- Validate `$ref` references (if schema uses `definitions` or `$defs`)
- Return structured validation errors: `{ valid: boolean, errors: ValidationError[] }`
- Each ValidationError: `{ path: string (JSON Pointer), message: string, code: string }`
- Support additionalProperties: false by default (reject unknown params), configurable per tool
- Support nullable fields per JSON Schema 2020-12
- Compile schema on tool registration, cache compiled validator
- Handle cyclic schemas gracefully (detect and reject with clear error)

### Non-functional
- Schema compilation: < 50 ms per schema (one-time cost)
- Validation time: < 5 ms for typical tool params
- Error message generation: < 2 ms

### Performance
- Compiled validators are cached in memory (Map keyed by toolName + schema hash)
- Schema compilation happens once on registry load and on each update

### Security
- Schema injection: validate that the input schema itself is valid JSON Schema before compilation
- Reject schemas that attempt to validate with custom keywords or functions (use only standard JSON Schema)
- Parameter size limit: max 1 MB serialized JSON for tool params
- No eval or dynamic code generation in schema compilation

### Accessibility
- N/A (internal validation)

## Architecture Constraints

- Use `ajv` with `strict: true` mode (disallows unknown keywords)
- Add `ajv-formats` for format validation (date-time, uri, email, etc.)
- Compiled validators are stored in a `Map<string, ValidateFunction>` and invalidated when tool schema changes
- The validator must handle `$ref` references within the same schema document (not external refs)
- Validation errors are formatted using ajv's `errors` array with our custom message mapping

## Acceptance Criteria

- Valid params pass validation with `{ valid: true, errors: [] }`
- Missing required field returns error with path and message
- Wrong type (string instead of number) returns error with expected type
- String exceeding maxLength returns error with limit
- Number below minimum returns error with limit
- Array exceeding maxItems returns error with limit
- Unknown field (additionalProperties: false) returns error
- Enum value not in allowed list returns error with allowed values
- Nested object validation returns errors with full JSON Pointer path
- Schema compilation error is caught and returned as validation error
- Validation completes in < 5 ms for typical params

## Notes

- ajv setup: `import Ajv from "ajv"; const ajv = new Ajv({ strict: true, allErrors: true });`
- Error format: `{ path: "/query/filters/date", message: "Must be a valid ISO 8601 date string", code: "INVALID_FORMAT" }`
- ajv error codes map to our codes: `required` → `MISSING_REQUIRED`, `type` → `WRONG_TYPE`, etc.
- Compiled validator caching:
  ```typescript
  const cache = new Map<string, ValidateFunction>();
  function getValidator(tool: ToolDefinition): ValidateFunction {
    const key = `${tool.name}:${JSON.stringify(tool.inputSchema)}`;
    if (!cache.has(key)) cache.set(key, ajv.compile(tool.inputSchema));
    return cache.get(key)!;
  }
  ```
- Use `ajv-formats` for `date-time`, `uri`, `email`, `uuid`, etc.
