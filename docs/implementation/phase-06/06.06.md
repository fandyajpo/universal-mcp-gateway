## Goal

Implement table extraction from PDF documents, detecting and extracting tabular data into structured JSON format (rows and columns). Tables are extracted using a combination of positional analysis (detecting aligned text blocks) and, optionally, a Python sidecar running Camelot-py for complex or borderless tables. Extracted tables are stored as structured data alongside the document for downstream use in RAG queries.

## Business Context

Enterprise documents — financial reports, data sheets, compliance filings — contain critical information in tables. Traditional text extraction loses the tabular structure, making table data effectively invisible to RAG retrieval. Table extraction preserves the row/column structure, enabling queries like "What was Q4 2024 revenue?" to return precise numerical data from tables within PDF reports.

## Technical Context

Table extraction is significantly harder than text extraction. Tables can be bordered (explicit grid lines), borderless (aligned whitespace), or mixed. The extraction approach uses a confidence-based cascade: first try PyMuPDF's built-in table detection (positional analysis), then fall back to a Python sidecar running Camelot-py for complex cases. Extracted tables are converted to JSON, CSV, and Markdown formats for downstream consumption.

## Requirements

### Functional
- Detect tables in PDF pages using positional analysis (aligned text, grid lines)
- Extract bordered tables (explicit lines) with high accuracy (> 95% cell accuracy)
- Extract borderless tables (whitespace-aligned) with good accuracy (> 85% cell accuracy)
- Output tables in three formats: JSON (array of objects), CSV string, Markdown table
- Preserve table headers and detect header row(s)
- Handle multi-page tables (merge across page boundaries)
- Handle merged cells (colspan, rowspan) — represent as null or merged indicator
- Detect table captions and associate them with the table
- Associate table with page number and location bbox for citation
- Extract cell-level metadata: text content, formatting (bold, italic), numeric detection
- Store extracted tables as a sub-array on the Document record
- Support table extraction confidence scoring and flag low-confidence cells

### Non-functional
- Extraction throughput: minimum 2 pages per second with table detection enabled
- Memory limit: 256 MB for Python sidecar process
- Table detection false positive rate < 5% (don't detect non-tables as tables)

### Performance
- Python sidecar has a 30-second timeout per document
- Table detection runs as part of extraction phase, not a separate step

### Security
- Python sidecar process runs with restricted filesystem access (temp directory only)
- Table content is sanitized before storage

### Accessibility
- N/A (internal processing service)

## Architecture Constraints

- Table detection runs on every page of the PDF during extraction
- Tables are extracted even if they appear inside a larger text block
- Multi-page tables are detected by matching column structure across consecutive pages
- The Python sidecar communicates via stdin/stdout JSON messages
- If Python/Camelot-py is unavailable, table extraction degrades to positional analysis only
- Tables with confidence < 70% are excluded from the primary table array but stored in a `lowConfidenceTables` field

## Acceptance Criteria

- Bordered tables extracted with > 95% cell-level accuracy
- Borderless tables extracted with > 85% cell-level accuracy
- Table output available in JSON, CSV, and Markdown formats
- Multi-page tables merged into a single logical table
- Merged cells correctly represented
- Table captions detected and associated
- Confidence below 70% stores table in low-confidence field
- Table extraction does not significantly slow overall pipeline (< 2x extraction time with tables vs without)

## Notes

- Camelot-py is the recommended Python library for table extraction — it handles both lattice (bordered) and stream (borderless) tables
- The Python sidecar script (`scripts/table-extract.py`) should accept a PDF path and page range, return JSON via stdout
- For bordered tables, grid line detection via PyMuPDF's `get_image_info()` works well
- For borderless tables, column detection uses whitespace clustering (find vertical alignment of text blocks)
- Table normalization: strip whitespace, normalize null values, detect numeric columns
