## Goal

Implement the PDF text extraction service that extracts text content from native PDFs using pdf.js (client-side preview) and PyMuPDF (server-side extraction). The service must preserve positional metadata (page number, bounding box, font size, font name) for downstream section detection and chunking. For scanned PDFs, the extraction must detect low text density and delegate to the OCR module (Step 06.03).

## Business Context

Enterprises deal with both digital-born PDFs (contracts, reports) and scanned documents (handwritten forms, signed agreements). The text extraction service must handle both types transparently. Extraction quality directly impacts the downstream RAG pipeline — poor extraction means poor retrieval. Positional metadata enables smart chunking (Step 06.04) that respects document structure.

## Technical Context

The extraction service runs as an Inngest step function triggered by the `pdf/extract/started` event. It reads the PDF from R2 using the signed URL, performs extraction, writes the extracted text and metadata to MongoDB, and emits `pdf/extract/completed` (or `pdf/extract/failed`). PyMuPDF (fitz) provides high-quality native extraction with positional data. pdf.js is used for client-side preview only. This step focuses on the server-side extraction.

## Requirements

### Functional
- Read PDF bytes from R2 given an R2 key
- Extract all text content with page numbers and bounding boxes
- Extract font metadata (name, size, weight, color) per text span
- Detect document language (heuristic based on character analysis)
- Handle PDFs with missing or corrupted font encodings
- Extract text in reading order (respects multi-column layouts when possible)
- Detect scanned/image-only PDFs (text density < 10 chars per page average)
- Return extracted content in a structured format: `{ pages: [{ pageNumber, width, height, blocks: [{ bbox, lines: [{ bbox, text, font }] }] }] }`
- Write extracted text to MongoDB Document record as `processedContent.text`
- Write extraction metadata (char count, page count, extraction method, confidence score) to Document record

### Non-functional
- Extraction throughput: minimum 5 pages per second for native PDFs
- Memory limit: 256 MB for extraction process (PyMuPDF loads entire PDF into memory)
- Extraction timeout: 5 minutes per document
- Accuracy target: > 95% character accuracy for native PDFs with standard fonts

### Performance
- Parallel extraction across documents: limited to 3 concurrent extractions per worker
- Large document splitting: documents > 500 pages are processed in 100-page batches

### Security
- PDF content never written to disk logs (log only page count, char count)
- Extraction runs in isolated process (worker thread for PyMuPDF)
- Memory-mapped file reading to avoid OOM for large PDFs

### Accessibility
- N/A (internal processing service)

## Deliverables

- `extractText()` function that returns structured extraction result
- Integration with PyMuPDF via child process or worker thread
- Scanned document detection heuristic
- Extraction result writing to MongoDB Document record
- Error handling for corrupted or malformed PDFs

## Architecture Constraints

- PyMuPDF runs in a child process for fault isolation (crash doesn't bring down the worker)
- Extraction method is recorded (`native` vs `ocr`) based on text density detection
- Corrupted font encodings fall back to character-by-character extraction
- Multi-column layout: detected by analyzing x-coordinate distribution of text blocks

## Acceptance Criteria

- Native PDF text extraction returns all visible text with correct page assignment
- Extraction preserves reading order for single-column documents
- Font metadata (name, size) correctly extracted for embedded fonts
- Scanned PDFs are correctly detected and flagged for OCR (text density < 10 chars/page)
- Corrupted font PDFs fall back gracefully without crashing
- Extraction metadata written to MongoDB includes char count, page count, and method
- Documents > 500 pages are processed in batches
- Extraction timeout (5 min) kills the process and emits `pdf/extract/failed`

## Notes

- PyMuPDF (fitz) is the recommended library because it provides positional metadata natively
- If PyMuPDF cannot be installed as a native Node.js addon, use a Python sidecar process communicating via JSON stdout
- Text density heuristic: count non-whitespace characters per page; if average < 10, classify as scanned
- Store extracted text as a compressed field in MongoDB (use zlib compression if > 100 KB)
