# Step

Step ID: 04.03

Step Name: Connection Pooling Configuration

Estimated Time: 30 minutes

Priority: High

Difficulty: Easy

Dependencies: 00.08

---

## Goal

Configure optimal MongoDB connection pool settings: min/max pool size, socket timeout, server selection timeout, heartbeat frequency, and connection health checks. Document the configuration rationale and provide guidance for production tuning.

---

## Requirements

1. Configure Mongoose connection pool:
   - `minPoolSize`: 5 (maintain minimum connections)
   - `maxPoolSize`: 50 (max concurrent operations — adjust based on workload)
   - `socketTimeoutMS`: 45000 (close slow sockets)
   - `serverSelectionTimeoutMS`: 5000 (fail fast if no server available)
   - `heartbeatFrequencyMS`: 10000 (check server health every 10s)
   - `retryWrites`: true
   - `w`: majority (write concern)
   - `readConcern`: majority (read concern)
2. Configure connection health checks:
   - `healthCheck()` method runs `db.admin().ping()`
   - Connection pool stats exposed via `getPoolStats()` (active, idle, available connections)
3. Document pooling configuration in `docs/architecture/connection-pooling.md`.
4. Add `DATABASE_POOL_MIN` and `DATABASE_POOL_MAX` to config schema (`@repo/config`).

## Acceptance Criteria

1. Connection manager uses configured pool size.
2. `getPoolStats()` returns current pool metrics.
3. Health check passes on healthy connection.
4. Pool configuration documented with tuning guidance.
5. Pool config overrideable via environment variables.
