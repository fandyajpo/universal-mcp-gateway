# Step

Step ID: 04.05

Step Name: Seed Scripts

Estimated Time: 1 hour

Priority: Medium

Difficulty: Easy

Dependencies: 04.04

---

## Goal

Create seed scripts that populate the database with realistic development data: sample users, workspaces, documents, audit logs, and configuration records. Seeds are idempotent (can be re-run without creating duplicates) and include both minimal and comprehensive data sets.

---

## Requirements

1. Create `scripts/seed.ts` — CLI entry point:
   - `pnpm seed` — runs all seeds
   - `pnpm seed:reset` — drops all data and re-seeds
   - `pnpm seed:minimal` — just enough data to develop with (1 workspace, 3 users)
2. Seed data:
   - **Users**: admin@example.com (password: Admin123!), user1@example.com, user2@example.com, viewer@example.com.
   - **Workspaces**: "Acme Corp" (with all 4 users in different roles), "Starter Workspace" (single user).
   - **Documents**: 5 sample markdown documents with varying content types, statuses, and metadata.
   - **Audit logs**: 50 sample audit log entries spanning the last 30 days.
   - **Api keys**: 2 sample API keys (one active, one revoked).
   - **Sessions**: Active sessions for seeded users.
   - **Invitations**: 2 pending invitations, 1 accepted, 1 expired.
3. Idempotency: check for existing data before inserting (use `email` or `slug` as dedup key).
4. Factory functions in `scripts/seed/factories/` for generating realistic test data.
5. Console output showing progress and summary.

---

## Files To Modify

---

## Acceptance Criteria

1. `pnpm seed` populates database with all seed data.
2. `pnpm seed` is idempotent — re-running does not create duplicates.
3. `pnpm seed:reset` drops all data and re-seeds cleanly.
4. Seeded data is realistic (valid emails, sensible names, real-looking documents).
5. All seeded users can log in with their passwords.
6. Seeded workspace has realistic member structure with multiple roles.
7. Console output shows progress for each seed category.
