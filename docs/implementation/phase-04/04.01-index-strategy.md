# Step

Step Name: Index Strategy Implementation

Estimated Time: 2 hours

Priority: Critical

Difficulty: Hard

Dependencies: 00.08

---

## Goal

Create and deploy a comprehensive MongoDB indexing strategy covering all entity collections. Document every index with its purpose, fields, type, and expected query patterns. Ensure all indexes are defined in Mongoose schemas (not created manually) and deployed via migration scripts.

---

## Requirements

1. Analyze all query patterns from existing repositories (User, Workspace, Session, ApiKey, AuditLog, Document).
2. Create indexes for each collection:
   - **users**: `email` (unique), `name` (text), compound `isActive + lastLoginAt`, compound `emailVerified + isActive`.
   - **workspaces**: `slug` (unique), `ownerId`, compound `isActive + plan`, text `name + description`.
   - **sessions**: `token` (unique), compound `userId + isValid`, `expiresAt` (TTL index, expireAfterSeconds: 0), compound `workspaceId + lastActivityAt`.
   - **api_keys**: `keyHash` (unique, sparse), compound `workspaceId + isActive`, compound `expiresAt` (partial for non-expired).
   - **audit_logs**: compound `workspaceId + createdAt` (descending), compound `userId + createdAt`, compound `action + createdAt`, compound `entityType + entityId + createdAt`.
   - **documents**: compound `workspaceId + status`, compound `workspaceId + contentType`, compound `workspaceId + createdAt` (descending), text `title + description + tags`, compound `workspaceId + updatedAt`.
   - **invitations**: `token` (unique), compound `workspaceId + status`, `expiresAt` (TTL index).
3. Document each index in `docs/architecture/database-indexes.md`.
4. Create migration script to deploy all indexes.
5. Verify indexes with `mongoose.model(...).listIndexes()` in tests.
6. Add performance note: index creation should use `background: true` and be deployed during low-traffic windows.
