## Goal

Create a database migration framework with versioned, idempotent, and reversible migration scripts. The framework supports both schema changes (indexes, collections) and data migrations (transform existing records). Migrations are run via CLI and tracked in a `_migrations` collection.

---

## Requirements

1. Create `scripts/migrate.ts` — CLI entry point:
   - `pnpm migrate:up` — runs all pending migrations
   - `pnpm migrate:down` — rolls back the last migration
   - `pnpm migrate:create <name>` — creates a new migration file from template
   - `pnpm migrate:status` — lists applied/pending migrations
2. Migration file structure: `scripts/migrations/{timestamp}-{name}.ts` exporting `up()` and `down()` functions.
3. Migration tracking: `_migrations` collection stores `{ id, name, appliedAt, duration, checksum }`.
4. Idempotency: `up()` checks if the change already exists before applying.
5. Reversibility: every `up()` must have a corresponding `down()`.
6. Transaction support: wrap migration in MongoDB transaction if replica set is available.
7. Migration template when creating: includes timestamp, numbered prefix, boilerplate `up`/`down` functions.
8. Add migration scripts to root `package.json`.

## Acceptance Criteria

1. `pnpm migrate:create add-users-index` creates a timestamped migration file.
2. `pnpm migrate:up` runs all pending migrations and records them in `_migrations`.
3. `pnpm migrate:up` is idempotent — running twice does nothing.
4. `pnpm migrate:down` rolls back the last migration.
5. `pnpm migrate:status` shows "applied" for completed migrations.
6. Migration failures are reported with clear error messages and partial rollback.
