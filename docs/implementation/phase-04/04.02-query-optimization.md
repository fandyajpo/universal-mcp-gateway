# Step

Step ID: 04.02

Step Name: Query Optimization and Profiling

Estimated Time: 2 hours

Priority: High

Difficulty: Hard

Dependencies: 04.01

---

## Goal

Profile all database queries used across the platform, identify slow queries (execution time > 100ms), optimize them with compound indexes, covered queries, and aggregation pipeline optimizations. Implement query timing middleware and slow query logging.

---

## Requirements

1. Enable MongoDB profiler (slowOpThresholdMs: 100) on development database.
2. Collect query profiles for all common operations: user lookup, session verification, workspace list, member list, document search, audit log queries, invitation queries.
3. For each slow query:
   - Run `explain("executionStats")` to identify COLLSCAN vs IXSCAN
   - Add/modify compound indexes to cover the query
   - Convert to covered query where possible (projection only on indexed fields)
   - Rewrite aggregation pipelines to use `$match` early
4. Implement query timing middleware in `BaseRepository`:
   - Wrap every query with timing
   - Log queries taking > 50ms at `warn` level
   - Log queries taking > 200ms at `error` level
5. Implement `@repo/database` slow query log via `@repo/logger`.
6. Create query performance baseline document in `docs/architecture/query-performance.md`.

## Acceptance Criteria

1. All common queries use index scans (IXSCAN), not collection scans (COLLSCAN).
2. Query timing middleware logs slow queries at correct levels.
3. Covered queries return data from index only (no FETCH stage).
4. Query performance document shows baseline metrics for all common queries.
5. P95 query latency is under 50ms for all common queries.
