## Goal

Implement compliance reporting capabilities for SOC 2 and GDPR. Build automated evidence collection for SOC 2 controls, GDPR data subject request handling (access, delete, portability), and compliance dashboards for enterprise admins.

---

## Business Context

Enterprise customers require their vendors to maintain compliance certifications (SOC 2) and support regulatory requirements (GDPR). SOC 2 requires evidence of controls: access controls, change management, data protection, monitoring. GDPR gives data subjects rights to access, delete, and export their data. Compliance reporting automates the evidence collection and request handling processes, making audits manageable.

---

## Technical Context

SOC 2 evidence collection is automated by the platform — key controls are monitored and evidence is collected periodically. Evidence includes: access control logs, audit trails, change management records, data backup verification, incident response logs, and security monitoring data. Evidence is stored in an `evidence_records` collection with timestamps and supporting data.

GDPR subject request handling implements the standard workflow: (1) user submits request via form, (2) admin verifies identity, (3) system collects user data across all data stores, (4) admin reviews and approves response, (5) response is delivered to user. For deletion requests, the system purges user data per retention policies while preserving audit records.

## Requirements

### Functional requirements

- SOC 2 evidence collection: automated collection of evidence for key controls — access control (user authentication events, role changes, permission changes), change management (deployment records, configuration changes), data protection (encryption at rest verification, backup verification), monitoring (system health data, incident records, alert logs)
- Evidence scheduling: configurable collection frequency per control (daily, weekly, monthly)
- Evidence repository: store evidence with timestamp, control ID, description, supporting data, and status
- SOC 2 report generation: generate SOC 2 Type II report with control descriptions, evidence summaries, and test results
- GDPR access request: accept subject access request, verify identity, collect all personal data across system, package for delivery, track request status
- GDPR deletion request: accept deletion request, verify identity, identify all personal data, execute deletion per retention policies, confirm completion, preserve audit trail exemption
- GDPR portability request: collect user data in machine-readable format (JSON), provide download link with expiration
- Request dashboard: track all GDPR requests with status, timestamps, and action history
- Compliance overview dashboard: SOC 2 control status, evidence collection health, GDPR request metrics

### Non-functional requirements

- Evidence collection must not impact application performance (async collection)
- GDPR data collection must complete within 30 days (regulatory requirement)
- SOC 2 report generation must include evidence from the full audit period
- All compliance data must be stored immutably

### Performance requirements

- Evidence collection per control: < 5s (async)
- GDPR data collection for a typical user: < 30 minutes
- SOC 2 report generation: < 10 minutes
- GDPR response packaging: < 1 minute

### Security requirements

- Compliance data must be access-restricted to `admin:compliance` role
- GDPR request identity verification must be rigorous (two-factor verification)
- Data collected for GDPR must not include data from other users
- Compliance reports must not expose sensitive security details
- Evidence must be cryptographically signed to prevent tampering

### Accessibility requirements

- Compliance dashboards must use accessible data tables
- Report downloads must have accessible file names and descriptions

## Architecture Constraints

- SOC 2 evidence is stored in an append-only `evidence_records` collection — evidence is never modified
- Each evidence record is cryptographically signed with the compliance package's private key
- GDPR data collection queries all data repositories that store personal data
- GDPR deletion skips audit log entries (audit logs are exempt from deletion per regulatory requirements)
- GDPR response packages are stored in R2 with signed URLs (7-day expiration)
- SOC 2 report periods align with calendar months for consistency
