# Step

Step ID: 03.01

Step Name: Workspace Schema and Validation

Estimated Time: 30 minutes

Priority: Critical

Difficulty: Easy

Dependencies: 00.05

---

## Goal

Create Zod validation schemas for workspace-related inputs: create workspace form, update settings, member invitations, slug validation. These schemas live in `@repo/validation` alongside the auth schemas.

---

## Files To Create

- packages/validation/src/schemas/workspace.ts

Schemas: `createWorkspaceSchema` (name 2-100 chars, slug 3-63 chars lowercase-hyphens, optional description), `updateWorkspaceSchema` (partial of create), `workspaceSettingsSchema` (timezone, locale, feature flags), `memberRoleSchema` (workspaceId, userId, role), `invitationSchema` (email, role, message).

---

## Acceptance Criteria

1. Valid workspace data passes validation.
2. Invalid slug (uppercase, special chars, too short) is rejected.
3. All schemas infer correct TypeScript types.
