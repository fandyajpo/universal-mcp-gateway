---

## Goal

Implement the `WorkspaceService` — the business logic layer for workspace operations. This service orchestrates workspace CRUD, member management, invitation handling, and RBAC enforcement. All workspace operations go through this service to ensure consistency, validation, and authorization.

---

## Files To Create

- packages/auth/src/services/workspace-service.ts

Methods: `create(data, userId)` — creates workspace, adds creator as owner, emits event; `getById(workspaceId, userId)` — returns workspace if user is member; `update(workspaceId, data, userId)` — validates ownership/admin; `archive(workspaceId, userId)` — soft delete with admin check; `listUserWorkspaces(userId)` — returns all workspaces user belongs to; `addMember(workspaceId, email, role, inviterId)` — adds existing user by email; `removeMember(workspaceId, userId, requesterId)` — admin/owner only; `transferOwnership(workspaceId, newOwnerId, currentOwnerId)` — owner only; `updateSettings(workspaceId, settings, userId)` — admin only.

---

## Acceptance Criteria

1. `create` sets creator as `owner`, generates unique slug.
2. `getById` only returns data if user is a member.
3. `update` fails with 403 for non-admin members.
4. `addMember` adds existing user or sends invitation.
5. `removeMember` fails if target is the last owner.
6. `transferOwnership` swaps roles between old and new owner.
7. `listUserWorkspaces` returns all workspaces ordered by last activity.
