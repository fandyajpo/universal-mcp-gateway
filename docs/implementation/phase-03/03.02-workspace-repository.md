# Step

Step ID: 03.02

Step Name: Workspace Repository

Estimated Time: 1 hour

Priority: Critical

Difficulty: Medium

Dependencies: 00.09

---

## Goal

Extend the existing `WorkspaceRepository` from Phase 00 with additional query methods needed for the workspace service: member queries, settings updates, slug uniqueness checks, and workspace transfer operations.

---

## Files To Modify

- packages/database/src/repositories/workspace.ts

Add methods: `findBySlug(slug, excludeId?)` (with uniqueness check), `addMember(workspaceId, userId, role)`, `removeMember(workspaceId, userId)`, `updateMemberRole(workspaceId, userId, role)`, `getMembers(workspaceId, filters?)`, `transferOwnership(workspaceId, newOwnerId)`, `archive(workspaceId)`, `restore(workspaceId)`, `updateSettings(workspaceId, settings)`.

---

## Acceptance Criteria

1. `addMember` creates a WorkspaceMember document with correct role.
2. `removeMember` soft-deletes the membership.
3. `getMembers` returns paginated members with user details.
4. `transferOwnership` changes owner to another admin.
5. `archive` sets `deletedAt` timestamp.
6. `findBySlug` returns null for unused slugs.
7. All methods are tenant-scoped.
