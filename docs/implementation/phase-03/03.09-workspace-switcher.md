# Step

Step ID: 03.09

Step Name: Workspace Switcher UI

Estimated Time: 1 hour

Priority: Medium

Difficulty: Easy

Dependencies: 03.04, 01.01

---

## Goal

Build the workspace switcher component in the web app sidebar header — a dropdown that shows the current workspace (name + avatar) and a list of the user's other workspaces. Clicking a workspace switches the current context.

---

## Files To Create

- apps/web/src/components/layout/workspace-switcher.tsx
- apps/web/src/components/layout/workspace-switcher-item.tsx
- apps/web/src/hooks/use-workspace.ts (Zustand store for current workspace)
- apps/web/src/hooks/use-user-workspaces.ts (React Query hook)

## Files To Modify

- apps/web/src/components/layout/sidebar.tsx (integrate switcher at top)

---

## Acceptance Criteria

1. Switcher shows current workspace avatar and name.
2. Clicking opens dropdown with list of all user's workspaces.
3. Each workspace shows avatar, name, and member count.
4. Current workspace has a checkmark indicator.
5. Clicking another workspace switches context (updates Zustand store, redirects).
6. "Create workspace" option at bottom of dropdown.
7. Active workspace is persisted in localStorage and restored on reload.
8. Switcher is disabled when user has only one workspace.
