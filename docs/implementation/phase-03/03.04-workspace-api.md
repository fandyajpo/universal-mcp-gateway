# Step

Step ID: 03.04

Step Name: Workspace API Routes

Estimated Time: 1 hour

Priority: High

Difficulty: Medium

Dependencies: 03.03

---

## Goal

Create Next.js API routes (Route Handlers) for workspace operations. These routes validate input with Zod, authenticate via middleware, authorize with RBAC, and return consistent JSON responses.

---

## Files To Create

- apps/web/src/app/api/workspaces/route.ts (POST — create, GET — list user's workspaces)
- apps/web/src/app/api/workspaces/[workspaceId]/route.ts (GET, PATCH, DELETE)
- apps/web/src/app/api/workspaces/[workspaceId]/members/route.ts (GET, POST)
- apps/web/src/app/api/workspaces/[workspaceId]/members/[userId]/route.ts (PATCH, DELETE)
- apps/web/src/app/api/workspaces/[workspaceId]/transfer/route.ts (POST)
- apps/web/src/app/api/workspaces/[workspaceId]/settings/route.ts (PATCH)

---

## Acceptance Criteria

1. `POST /api/workspaces` creates a workspace and returns it with 201.
2. `GET /api/workspaces` returns user's workspaces.
3. `GET /api/workspaces/:id` returns workspace details (members only).
4. `PATCH /api/workspaces/:id` updates workspace (admin only).
5. `DELETE /api/workspaces/:id` archives workspace (admin only).
6. `POST /api/workspaces/:id/members` adds member (admin only).
7. `DELETE /api/workspaces/:id/members/:userId` removes member (admin only).
8. `POST /api/workspaces/:id/transfer` transfers ownership (owner only).
9. All routes return consistent JSON: `{ data, error }` shape.
10. All routes validate input with Zod.
11. All routes are authenticated (401 if no session) and authorized (403 if wrong role).
