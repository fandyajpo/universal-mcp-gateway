## Goal

Run the complete verification suite for Phase 17: Performance Optimization. Execute all performance tests, verify correctness of optimizations, and produce a comprehensive before/after performance benchmark report demonstrating measurable improvements.

---

## Business Context

Performance optimization is the most measurable phase — every optimization must have a before/after metric. This verification step is not just about passing tests but demonstrating tangible improvements: faster page loads, lower API latency, reduced database query time, and better Lighthouse scores. The performance benchmark report is the key deliverable that proves the value of this phase.

---

## Technical Context

Verification combines functional testing (no regressions from optimizations) with performance benchmarking. The benchmark compares key metrics before and after the optimizations: Lighthouse scores, API response times, database query times, bundle sizes, and CDN cache hit ratios. The benchmark is run against a dedicated staging environment with production-like data volume.

## Requirements

### Functional requirements

- Run all unit tests from steps 17.01-17.08 and verify 100% pass rate
- Run Lighthouse CI against all apps and verify scores meet budgets
- Run k6 smoke and load tests against all scenarios
- Verify no functional regressions caused by optimizations
- Measure and report: bundle size comparison (before/after), Lighthouse score comparison, API p95 latency comparison, database query p95 comparison, CDN cache hit ratio, First Contentful Paint improvement, Time to Interactive improvement, page weight reduction
- Verify that all optimizations work together without conflicts
- Test with production-like data volume (100k+ users, 1M+ documents)

### Non-functional requirements

- Performance benchmark must be repeatable (same results within 5% variance)
- Lighthouse tests must run 3 times and use the median score
- k6 tests must run against a staging environment with representative data
- All benchmarks must be documented with methodology and environment details

### Performance requirements

- Lighthouse performance score: >= 95 on all routes
- FCP: < 1.5s
- LCP: < 2.5s
- TTI: < 3.5s
- API p95: < 200ms
- Database query p95: < 100ms
- Bundle size (initial JS): < 200KB

### Security requirements

- Performance benchmarks must not expose sensitive data
- Load testing must not impact production

### Accessibility requirements

- Lighthouse accessibility scores must be >= 95
