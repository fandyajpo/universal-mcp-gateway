## Goal

Optimize all images across the platform using Next.js Image component, Cloudflare Image Resizing, responsive images, lazy loading, and proper format selection (WebP, AVIF). Reduce image payload by 60%+.

---

## Business Context

Images are often the largest assets on a web page. Unoptimized images increase page load time, consume bandwidth, and hurt Core Web Vitals (LCP, CLS). With the platform's document previews, user avatars, marketplace screenshots, and branding logos, image optimization has significant impact on perceived performance.

---

## Technical Context

Next.js `<Image>` component provides automatic optimization: resizing, format conversion (WebP/AVIF), lazy loading, and responsive srcset generation. Cloudflare Image Resizing provides additional server-side transformation. The optimization covers: user avatars, workspace logos, marketplace listing icons and screenshots, PDF page previews, document thumbnails, and any other user-uploaded images.

## Requirements

### Functional requirements

- Replace all `<img>` tags with Next.js `<Image>` component
- Configure `next.config.ts` with allowed image domains (R2, gravatar, etc.)
- Set explicit `width` and `height` on all images to prevent Cumulative Layout Shift (CLS)
- Implement responsive images with `sizes` attribute for different viewports
- Enable automatic format conversion (AVIF with WebP fallback)
- Configure quality settings per image type (85% for photos, 90% for screenshots)
- Implement lazy loading for below-the-fold images
- Add `loading="eager"` for above-the-fold hero images (LCP optimization)
- Configure Cloudflare Image Resizing for dynamically resized images
- Add blur placeholder for images using `placeholder="blur"` with `blurDataURL`

### Non-functional requirements

- All images must use Next.js Image component — no unoptimized `<img>` tags
- Image format must be AVIF (primary) with WebP fallback for browser compatibility
- Image CDN must have Cache-Control: `public, max-age=31536000, immutable`
- Lazy loaded images must use `loading="lazy"` attribute

### Performance requirements

- Image payload reduction: 60%+ from baseline
- LCP image load: < 1.5s
- No CLS from images (explicit dimensions on all images)
- Time to first image paint: < 1s

### Security requirements

- Allowed image domains must be explicitly configured — no wildcard allowlist
- User-uploaded images must be served from a controlled domain
- Image URL sanitization to prevent open redirect

### Accessibility requirements

- All images must have `alt` text
- Decorative images must have `alt=""` (empty alt)
- Image captions must be associated with images

## Architecture Constraints

- Next.js `images.domains` configuration is deprecated — use `images.remotePatterns` instead
- Allowed remote patterns: Cloudflare R2 bucket, Gravatar, and any other image sources
- Cloudflare Image Resizing is configured via URL parameters on R2 image URLs
- Explicit width/height must be provided for all images — no automatic dimension detection
- Images stored in R2 should be served through Cloudflare's CDN with image resizing

## Notes

The biggest gains come from format conversion (JPEG/PNG → AVIF/WebP) and proper sizing (never serve a 2000px image in a 200px container). Use Chrome DevTools' Coverage tab to identify images that are loaded but not displayed. The Next.js Image component handles most optimization automatically — the key work is migrating existing `<img>` tags and adding proper configuration. Marketplace screenshot galleries should use a dedicated optimized Image component with thumbnail/preview pattern.
